## 候选全集清单

- 生图类型：页面设计稿
- 首选候选：页面线框 / artboard
- 必生成备选候选：
  - 多候选 artboard filmstrip
  - 响应式状态板
  - 主路径页面流
- 校准期要求：当前图型命中后，HTML 内必须按候选 A/B/C/D…纵向展开首选候选与全部备选候选；每个候选都必须是真图。信息不足时也必须生成该备选候选，用“待确认节点”标明缺口。

## 页面设计稿规则

生成页面设计 HTML 前先做 5 个极短判断：

1. 唯一核心目标是什么？
2. 用户当前最需要完成什么动作？
3. 首层必须展示什么？
4. 哪些内容应收纳到详情、抽屉、折叠区、二级 Tab 或更多菜单？
5. 是否存在卡片堆叠、标题废话、功能平铺？如何规避？

页面设计稿用于方向评审时，默认在一个单文件 HTML 内提供 3 个候选设计稿；当用户明确要求多方案、视觉方向开放、或需要覆盖明显不同的信息架构时，可扩展到 4-5 个。候选以 A/B/C/D/E artboard 在同一张“页面设计稿”主图内呈现，不要拆成多个 HTML 文件，也不要混入其他图型。

每个候选必须围绕同一个用户目标和同一条主路径，而不是把不同功能页或需求清单拆成多个页面。每个候选必须静态外显：

- `visual thesis`：一句话说明视觉情绪、材质和能量。
- `content plan`：首屏、支撑、细节、行动或任务闭环。
- `interaction thesis`：2-3 个关键交互或动效方向，并标注 reduced-motion 降级。
- 首层展示、收纳区、主 CTA 或主操作。
- 默认、加载、空、错误态，至少以缩略状态、旁注或状态条可见。
- 移动端、平板、桌面响应式口径，以及键盘焦点、对比度、点击目标、非颜色状态表达等可访问性约束。

候选之间必须有本质差异：至少在信息架构、首屏视觉锚点、交互递进、布局节奏或响应式结构中有一项明显不同；禁止只换颜色、阴影、圆角凑数量。

多个候选的排版必须按被设计对象选择：

- Web 端候选稿优先纵向排列，用页面长度保留每个候选的真实首屏比例、滚动节奏和关键状态；不要为了并排对比把每个网页稿压成不可读缩略图。
- 移动端设计稿可以在桌面宽度下用横向手机稿 filmstrip 对比，因为手机稿较窄，横向比较更容易看出导航、首屏和操作差异。
- 真实移动端查看时不得依赖横向溢出理解候选；必须提供可访问的上一稿/下一稿、scroll-snap 或纵向降级，保证 390px
  宽度下不点击、不横向拖动也能读到候选摘要、推荐项和关键差异。

HTML 内必须提供候选对比矩阵和推荐项，说明每个候选的优点、缺点、适用边界、调整成本和推荐理由。推荐项要突出，但不能把其他候选画成错误方案。

用户明确指定只要一个最终稿、已有方向已确认、或本轮属于缺陷修复或最终交付时，可以只输出单稿；但必须在 HTML 内说明“不发散候选”的理由。

页面只能服务一个主路径；禁止把需求列表直接翻译成卡片堆叠；必须体现默认、加载、空、错误态，并覆盖移动端、平板、桌面。

## 多方案 / 多候选表达规则

校准期多候选不再是页面设计稿专属；每次只对当前命中的生图类型生成候选全集，后续用户确认首选后再恢复为“单首选 + 按需备选”。当前候选表必须保持：

- 系统架构图：首选北向南分层拓扑，备选主请求中轴 + 控制/数据/兜底泳道、运行时依赖拓扑。
- 业务架构图 / 领域地图：首选能力层 + 领域对象关系图，备选参与方边界图、规则约束热区图、价值链地图。
- 业务流程图：首选 BPMN-light 流程图，备选泳道流程图、阶段轨道图、异常分支流程图。
- 代码时序图：首选参与者列 + 时间向下时序图，备选异步回调时序图、事务边界时序图、重试/异常返回时序图。
- 状态 / 数据模型图：首选状态机图，备选 ER-lite、生命周期轨道、数据流图、状态-事件矩阵热区。
- 故障排查图：首选排障时序图，备选因果链图、BPMN-light 排查流程、before/after 流程化对照、状态/数据断点图。
- 功能迭代 / 开发设计图：首选当前流程 vs 目标流程的流程化对照，备选 current/target 技术时序、差异热区、发布回滚轨道。
- 页面设计稿：首选页面线框 / artboard，备选多候选 artboard filmstrip、响应式状态板、主路径页面流。
- 技术设计图：首选模块 / 契约 / 数据 / 发布回滚拓扑，备选 API 契约泳道、数据流 + 一致性边界、发布切换轨道。
- 需求 / 决策沟通图：首选决策树，备选方案矩阵 + 主路径绑定、取舍象限、推荐路径图。
- 交付验收图：首选需求到证据的验收轨道，备选 R# 泳道验收看板、证据矩阵热区、地铁站点式验收线路。

多候选排版默认规则：

- Web 端多候选优先纵向展开，让每个候选保持可读比例、关键路径和状态外显；不要把候选横向压成缩略图。
- 窄移动稿可在桌面视口横向 filmstrip 对比；但真实移动端不得把横向滚动作为唯一阅读路径，必须提供纵向降级、锚点导航或上一稿/下一稿控制。
- 每个候选必须有明确差异维度、适用边界、推荐理由和回滚或调整成本；禁止只换颜色、阴影、圆角凑数量。
