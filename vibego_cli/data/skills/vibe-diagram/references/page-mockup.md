## 页面设计稿规则

生成页面设计 HTML 前先做 5 个极短判断：

1. 唯一核心目标是什么？
2. 用户当前最需要完成什么动作？
3. 首层必须展示什么？
4. 哪些内容应收纳到详情、抽屉、折叠区、二级 Tab 或更多菜单？
5. 是否存在卡片堆叠、标题废话、功能平铺？如何规避？

页面设计稿用于方向评审时，默认在一个单文件 HTML 内提供 3 个候选设计稿；当用户明确要求多方案、视觉方向开放、或需要覆盖明显不同的信息架构时，可扩展到
4-5 个。候选以 A/B/C/D/E artboard 在同一张“页面设计稿”主图内呈现，不要拆成多个 HTML 文件，也不要混入其他图型。

每个候选必须围绕同一个用户目标和同一条主路径，而不是把不同功能页或需求清单拆成多个页面。每个候选必须静态外显：

- `visual thesis`：一句话说明视觉情绪、材质和能量。
- `content plan`：首屏、支撑、细节、行动或任务闭环。
- `interaction thesis`：2-3 个关键交互或动效方向，并标注 reduced-motion 降级。
- 首层展示、收纳区、主 CTA 或主操作。
- 默认、加载、空、错误态，至少以缩略状态、旁注或状态条可见。
- 移动端、平板、桌面响应式口径，以及键盘焦点、对比度、点击目标、非颜色状态表达等可访问性约束。

候选之间必须有本质差异：至少在信息架构、首屏视觉锚点、交互递进、布局节奏或响应式结构中有一项明显不同；禁止只换颜色、阴影、圆角凑数量。

多个候选的排版必须按被设计对象选择：

- Web 端候选稿优先纵向排列，用页面长度保留每个候选的真实首屏比例、滚动节奏和关键状态；不要为了并排对比把每个网页稿压成不可读缩略图。
- 移动端设计稿可以在桌面宽度下用横向手机稿 filmstrip 对比，因为手机稿较窄，横向比较更容易看出导航、首屏和操作差异。
- 真实移动端查看时不得依赖横向溢出理解候选；必须提供可访问的上一稿/下一稿、scroll-snap 或纵向降级，保证 390px
  宽度下不点击、不横向拖动也能读到候选摘要、推荐项和关键差异。

HTML 内必须提供候选对比矩阵和推荐项，说明每个候选的优点、缺点、适用边界、调整成本和推荐理由。推荐项要突出，但不能把其他候选画成错误方案。

用户明确指定只要一个最终稿、已有方向已确认、或本轮属于缺陷修复或最终交付时，可以只输出单稿；但必须在 HTML 内说明“不发散候选”的理由。

页面只能服务一个主路径；禁止把需求列表直接翻译成卡片堆叠；必须体现默认、加载、空、错误态，并覆盖移动端、平板、桌面。

## 多方案 / 多候选表达规则

多候选不是页面设计稿的专属排版问题，但 3-5 个默认候选只适用于页面设计稿方向评审。其他图型只有在用户明确要求“多个方案 /
多个候选 / A/B/C 对比”，或该任务本身是方案选择时，才发散多个候选；否则保持单主图、单主路径。

逐图型适用边界：

- 系统架构图：默认单架构；只有用户要求架构方案对比时才输出 2-3 个候选架构，并对比部署边界、中间件、队列语义、状态面、风险和回滚。
- 业务架构图 / 领域地图：默认单领域地图；多候选只用于领域边界、能力分层或角色协作方案选择，并对比职责归属、规则口径、上下游影响。
- 业务流程图：默认单主流程；多候选流程必须用流程方案 A/B/C、泳道或阶段对照表达，并对比主路径步数、异常分支、状态回退、人工介入点。
- 代码时序图：默认单调用链；多候选只用于替代调用策略、事务边界、重试/异步策略对比，并对比参与者、时序、幂等、异常返回和事务边界。
- 状态 / 数据模型图：默认单模型；多候选只用于状态机、实体边界、索引或迁移策略对比，并对比约束、字段口径、迁移成本、一致性风险和回滚。
- 故障排查图：不生成 3-5 个修法设计稿；多假设必须进入假设裁决，修法只保留当前推荐动作、验证动作和回滚动作，避免把未验证修法画成可选设计稿。
- 页面设计稿：方向评审默认 3 个候选，可扩展到 4-5 个；候选必须是同一目标、同一主路径下的不同信息架构、视觉锚点或交互递进。
- 技术设计图：方案对比保持 2-4 个，必须包含最小改造、推荐改造，必要时加入行业最佳或保守回滚方案。
- 需求 / 决策沟通图：方案对比保持 2-4 个，必须展示推荐项、反例、适用边界、验收标准和未决问题。

多候选排版默认规则：

- Web 端多候选优先纵向展开，让每个候选保持可读比例、关键路径和状态外显；不要把候选横向压成缩略图。
- 窄移动稿可在桌面视口横向 filmstrip 对比，但真实移动端不得把横向滚动作为唯一阅读路径；必须有候选摘要、可访问的上一项/下一项、scroll-snap
  或纵向降级。
- 每个候选必须有明确差异维度、适用边界、推荐理由和回滚或调整成本；禁止只换颜色、阴影、圆角、图标或文案顺序凑方案数量。
- 如果一个候选无法说明“何时优于其他候选”，应删除该候选，而不是继续保留低质量占位。
