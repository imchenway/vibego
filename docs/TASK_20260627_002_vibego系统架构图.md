- 图型：系统架构图。

## 3. 第 2 轮反馈修正：去除卡片堆砌

### 3.1 现象

用户反馈上一版系统架构图“不像话，纯粹是卡片堆砌”。复核 HTML 后确认：主画布使用多行 `<button class="node ...">` 网格承载组件，虽然有纵向连接器，但主要信息仍依赖逐块阅读文字，不是通过拓扑关系理解系统。

### 3.2 影响

- 第一眼看到的是同权重组件块，而不是系统入口、控制面、工作面、运行时、数据面的关系。
- `TaskService / CommandService / ParallelSessionStore`、`tmux / 模型 CLI / session_binder` 等关系被弱化为并排块。
- “无独立 MQ / Broker”边界没有真正落在线路关系中，容易误读。

### 3.3 根因

上一版虽选择了北向南分层，但实现层退化为 CSS grid + button 节点：

- 每层组件以等权重按钮平铺；
- 主连线只承担层间过渡，不承担组件间语义；
- 证据与结论区视觉重量偏高，形成“主图 + 说明块”的拼盘感。

### 3.4 修法

- 保持同一 HTML 路径，重写主画布为单张 SVG 分层拓扑。
- 主线用中央 spine 表达：Telegram/CLI → Master → run/stop → Worker → 模型 CLI → session JSONL。
- 旁路用虚线/曲线表达：ProjectRepository、StateStore、Task/Command/Parallel、tmux、session_binder、SQLite/logs。
- 组件改用架构符号：云、六边形、圆柱、终端、菱形、文档，不再使用按钮网格作为主图节点。
- 移动端单独提供纵向 SVG，避免整张桌面图等比缩小。
