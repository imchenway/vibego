# TASK_20260310_013 并行创建处理中消息成功后收口覆盖

## 1. 背景

用户反馈：

- Telegram 中点击 `开始并行处理` 后
- 流程消息被覆盖成：

```text
正在创建并行副本并启动并行 CLI，请稍候……
```

- 但后续即使 tmux / 并行会话已经成功运行
- 这条消息仍停留在“正在创建...”，看起来像卡住

## 2. 根因结论

- `bot.py:7590-7591`
  - 点击确认后，确实会先把当前流程消息改成“正在创建...”
- `bot.py:7635-7651`
  - 成功后没有继续编辑这条消息
  - 而是改为新增：
    - 推送预览
    - 并行副本摘要
    - session ack

所以并行会话实际成功了，但流程消息没有收口覆盖。

- `on_parallel_branch_confirm_callback(...)`
  - 成功后先继续覆盖当前流程消息为：
    - `已创建并行开发副本（原目录未改动）：...`
  - 然后再发送：
    - 推送预览
    - session ack

这样 Telegram 里不会一直停留在“正在创建...”
