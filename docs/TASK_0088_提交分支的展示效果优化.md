# /TASK_0088 提交分支的展示效果优化

## 1. 任务结论

- 目标：优化 Telegram 中提交/合并结果消息的可读性。
- 决策：三类结果消息统一为“总览 + 失败置顶 + 成功/跳过分组”的纯文本结构。
- 范围：
  - 原生会话提交结果
  - 并行分支提交结果
  - 自动合并结果
- 不改：提交/推送/自动合并逻辑、按钮布局、回调协议。

## 2. 关键证据

- 当前展示函数为平铺列表：`bot.py`（锚点：`def _format_parallel_operation_lines`）
- 三类结果消息共用该函数：
  - `bot.py`（锚点：`_format_parallel_operation_lines("分支提交结果：", combined_results)`）
  - `bot.py`（锚点：`_format_parallel_operation_lines("并行分支提交结果：", result.results)`）
  - `bot.py`（锚点：`title = "自动合并成功：" if not result.failed else "自动合并失败："`）
- 结果状态来源：`parallel_runtime.py`（锚点：`class RepoOperationResult`）
  - 已证实状态：`failed / skipped / committed / pushed / merged`

变更：

1. 重写 `_format_parallel_operation_lines`
   - 统一去掉标题末尾 `：/:`
   - 统计总数 / 失败 / 成功 / 跳过
   - 按 `failed -> success -> skipped` 分组输出
   - 多行 `message` 统一缩进到仓库名下方
2. 保持三类结果入口继续复用同一函数
3. 未修改 `parallel_runtime.py` 的结果判定和状态来源
