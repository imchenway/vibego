需求决策：可以独立，但应采用开放 Skill 核心与多客户端适配层

截至 2026-07-14，现有 vibe-diagram 已具备完整内容资产和合法 Codex plugin 结构；它不是从零开始。 但当前发布、版本、许可证、交付语义和验证仍与 Vibego / Codex / Telegram 耦合，因此准确口径是：具备抽取基础，尚未达到跨客户端成熟发布状态。

目标：一个制图能力，多端可安装、可验证、可升级 支持 Codex、Claude Code、Gemini CLI、GitHub Copilot CLI，以及其他 Agent Skills 兼容客户端。
内容基础成立
现有核心已经可复用 209 行薄内核、11 个按需 reference、58 个 HTML 模板、1 个纯标准库 linter；模板与插件镜像已有回归保护。
判断：不需要重写 应做“抽取 + 标准化 + 多端包装”，保留已验证的图型规则、模板与 lint。
结构部分成立
已有 Codex 分发，但不是通用分发 .codex-plugin/plugin.json 与 repo marketplace 已通过；Claude 校验当前目录明确失败，Gemini/Copilot 也没有各自 manifest 与调用 smoke。
判断：共用 skill，不共用 manifest Agent Skills 核心可跨端；安装目录、插件 manifest、调用语法、权限和预览交付必须由宿主适配。
成熟发布未达标
独立产品门禁仍缺失 版本与 Vibego 锁步；根 Apache-2.0 与 manifest proprietary 冲突；无跨客户端 eval / 安装 / 调用 / 升级矩阵；wheel 还携带被跟踪的 CPython 3.11 缓存。
判断：现在不能宣称“成熟跨端” 许可证先 fail-closed；其余能力需用跨端可复现证据逐项升级，不能用“目录能被读到”代替运行验证。
推荐路径：唯一 canonical core + 能力型交付契约 + 多客户端包装 核心只定义如何选图、生成自包含 HTML、校验质量；Codex / Claude / Gemini / Copilot / Telegram 分别负责安装、调用、权限、附件或本地链接交付。Vibego 改为消费已发布核心，而不是继续作为版本与分发源。

最小成熟化边界

“跨客户端”不应等同于制造一个所有工具都识别的万能 manifest；成熟做法是稳定共享标准核心，把宿主差异限制在薄适配层。

层 保留 / 建立 禁止混入 验收信号

客户端兼容口径：共享核心，分别包装

下表基于各厂商截至 2026-07-14 的官方文档；“支持”只表示协议与安装入口存在，不代表当前 Vibego 仓库已做完端到端验证。

客户端 共享 Skill 入口 平台包装 当前仓库状态

GitHub Copilot CLI ~/.agents/skills / .agents/skills 等官方目录 plugin.json；skills / agents / hooks / MCP / LSP 可打包 通用目录具备；未做正式 plugin 与调用 smoke

只有通过这些门禁，才可称为“成熟 skill”

这些是成熟度验收条件，不是已获 develop 许可的实现计划。

  1. 唯一事实源核心 skill 只维护一份；各插件包由发布流程生成或校验字节一致。
  2. 许可与版本独立裁决 Apache-2.0 与 proprietary 冲突，并建立独立 semver / changelog / rollback。
  3. 规范与静态质量skills-ref validate、四端 manifest validator、58 模板资产完整性与 HTML lint 全绿。
  4. 跨模型 eval每类图至少有真实 prompt；每端同时跑 with-skill、without-skill / previous-version 基线。
  5. 端到端安装与调用干净环境完成 discover → install → explicit/implicit invoke → 生成文件 → 打开/附件交付 → upgrade / rollback。
  6. 视觉与可访问性桌面、390px、无 JS、reduced-motion、键盘焦点、无横向溢出；关键结论不依赖弹窗。
  7. 跨 OS 与依赖边界macOS / Linux / Windows；无第三方依赖路径优先，脚本不可用时有纯文本 HTML fallback。
  8. 支持承诺可审计README 维护明确 compatibility matrix：verified / experimental / unsupported，不做模糊“全平台支持”宣传。
发布阻塞：当前 LICENSE:1-5 是 Apache-2.0，但 pyproject.toml:14 与 plugins/vibe-diagram/.codex-plugin/plugin.json:28 声明 LicenseRef-Proprietary。 在用户明确选择开源公共发布或私有/商业发布之前，必须 fail-closed，不能发布独立包。
