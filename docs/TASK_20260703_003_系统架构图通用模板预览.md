# TASK_20260703_003 系统架构图通用模板预览

## 目标

基于用户确认“对味”的复刻版抽象出通用模板预览。模板只保留视觉语法，不固化内部业务内容、emoji、节点文案或 Agent/RAG/LLM 等特定架构。

## 2026-07-03 反馈修正

用户指出第一版模板内部 emoji、文案和架构限制仍然太死，不是每个系统都一样。本次改成“可变槽位版”：

1. 图标从固定 emoji 改为 `icon` 槽；生成时可选择 emoji、SVG 小图标、字母徽标或留空。
2. 区域标题从“接入网关 / 核心应用 / Agent / 能力支撑”等半业务词，改为 `{{region.title}}` 和 `{{group.title}}` 槽。
3. 入口 rail、侧栏、支撑层、底座均标记为可隐藏 / 可改名，不再假设每个系统都有同样结构。
4. 保留的硬约束只剩视觉质量：宽画布、清晰边界、节点居中、箭头有源有目标、无编号实现分层、无接口流水线标题。

## 模板边界

- 固定：宽画布、边界层次、箭头类型、图例、节点居中、信息密度门禁。
- 可替换：所有标题、节点名、图标、分区名、分区显隐、节点数量、箭头标签。
- 禁止：把主画布标题改成接口调用链、使用 `1.` / `2.` / `3.` 编号实现分层、把业务源码路径写进主图节点。

- 已把右侧栏从固定业务含义改为 `旁路 / 管理/运维 / 治理槽位`，可隐藏、可改名、可拆分。
- 图例改为 `主请求流 / 主关系`、`支撑/数据流`、`旁路/例外/约束`，只描述线型语法，不绑定具体系统形态。
- 模板继续保留无固定 emoji、无固定业务节点、无固定 Agent/RAG/LLM 结构的可变槽位表达。
