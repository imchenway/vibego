# TASK_20260529_006 新增微信手机自动预览命令

## 1. 背景与目标

用户希望在现有命令管理中，除了 `wx-dev-preview`（生成微信开发预览二维码）之外，再新增一个命令用于触发微信开发者工具的“手机自动预览”。

用户最终选择命令名：`wx-auto-preview`。

目标口径：

- 新增默认通用命令 `wx-auto-preview`。
- 调用微信开发者工具 CLI：`auto-preview`。
- 复用现有微信小程序目录选择、端口配置、端口缺失/不匹配可恢复重试链路。
- 不生成二维码、不回传图片；仅返回命令执行摘要与详情入口。

## 2. 现状证据

- 默认通用命令位于 `command_center/defaults.py`，已有 `wx-dev-preview` 与 `wx-dev-upload`。
- 微信命令目录选择与端口修复由 `bot.py` 中 `_is_wx_devtools_command`、`_maybe_handle_wx_preview`、`_apply_wx_preview_port_and_retry` 统一处理。
- `scripts/gen_preview.sh` 已具备小程序目录探测、端口映射读取、端口错误提示和微信 CLI `preview` 调用能力。
- 本机微信开发者工具 CLI 已确认支持 `auto-preview --project --info-output --port`。

## 3. 方案对比

| 方案 | 内容 | 优点 | 缺点 | 结论 |
|---|---|---|---|---|
| A | 新增 `wx-auto-preview`，复用现有微信目录/端口链路，脚本增加 `auto-preview` 模式 | 体验一致、复用端口修复、改动可控 | 需要改脚本/默认命令/bot/测试 | 采用 |
| B | 只新增裸命令直接调 CLI | 改动最少 | 不能复用目录选择和端口修复，失败体验差 | 不采用 |
| C | 新建完全独立脚本复制所有逻辑 | 隔离强 | 复制大量逻辑，维护成本高 | 不采用 |

## 4. 开发设计

### 4.1 受影响目录

| 文件 | 变更 |
|---|---|
| `command_center/defaults.py` | 新增默认通用命令 `wx-auto-preview`。 |
| `bot.py` | 新增命令常量，将 `wx-auto-preview` 纳入微信开发者工具命令集合；补充目录选择文案、重试提示、项目目录解析。 |
| `scripts/gen_preview.sh` | 增加 `WX_PREVIEW_ACTION=auto-preview` 模式，调用 CLI `auto-preview`，不输出 `TG_PHOTO_FILE`。 |
| `command_center/fsm.py` | 将微信 FSM 注释从单一 preview 调整为微信开发者工具命令通用流程。 |
| `tests/test_wx_preview_detection.py` | 覆盖默认命令与 bot 命令识别。 |
| `tests/test_wx_preview_port_flow.py` | 覆盖脚本 auto-preview 模式。 |
| `README.md` | 更新微信开发命令端口说明。 |
| `AGENTS.md` | 更新微信开发命令事实表。 |

### 4.2 契约变更

| 契约 | 修复前 | 修复后 |
|---|---|---|
| 默认通用命令 | `wx-dev-preview`、`wx-dev-upload` | 增加 `wx-auto-preview` |
| 手机自动预览 | 无内置命令 | `wx-auto-preview` 调用 `cli auto-preview` |
| 二维码回传 | `wx-dev-preview` 输出 `TG_PHOTO_FILE` 并回传图片 | `wx-auto-preview` 不输出 `TG_PHOTO_FILE`，不回传图片 |
| 端口处理 | preview/upload 复用端口配置与可恢复重试 | auto-preview 同样复用 |
| 数据库 | 无 schema 变化 | 无 schema 变化；新命令通过 `vibego commands-seed` 注入缺失默认命令 |

### 4.3 关键流程

1. 用户在命令管理点击或触发 `wx-auto-preview`。
2. bot 识别为微信开发者工具命令，扫描当前工作目录及一层子目录的小程序候选。
3. 用户选择小程序目录。
4. bot 注入 `PROJECT_PATH/PROJECT_BASE` 后执行默认命令。
5. `scripts/gen_preview.sh` 在 `WX_PREVIEW_ACTION=auto-preview` 下：
   - 解析小程序目录；
   - 解析 IDE HTTP 服务端口；
   - 调用 `cli auto-preview --project <path> --info-output <file> --port <port>`；
   - 成功时输出“手机自动预览已触发”，不输出 `TG_PHOTO_FILE`。
6. 若端口缺失或不匹配，沿现有可恢复链路提示输入/选择端口并重试。

## 7. 风险与回滚

风险：

1. 微信开发者工具 `auto-preview` 要求本机 IDE 已登录，且手机侧具备自动预览条件；vibego 只能反馈 CLI 执行结果。
2. 如果本机 IDE 服务端口与配置不一致，仍会进入现有端口修复流程。
3. 既有 `gen_preview.sh` 增加模式分支，需确保默认 `preview` 行为不变。

回滚：

1. 删除 `command_center/defaults.py` 中 `wx-auto-preview` 默认命令。
2. 从 `bot.py` 中移除 `WX_AUTO_PREVIEW_COMMAND_NAME` 与相关文案/识别分支。
3. 删除 `scripts/gen_preview.sh` 中 `WX_PREVIEW_ACTION=auto-preview` 分支。
4. 回滚 README、AGENTS 与新增测试。
5. 若已执行 `vibego commands-seed` 注入，需要在命令管理或数据库中删除 `wx-auto-preview` 命令记录。
