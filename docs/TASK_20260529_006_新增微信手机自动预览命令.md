# TASK_20260529_006 新增微信手机自动预览命令

## 1. 背景与目标

用户希望在现有命令管理中，除了 `wx-dev-preview`（生成微信开发预览二维码）之外，再新增一个命令用于触发微信开发者工具的“手机自动预览”。

用户最终选择命令名：`wx-auto-preview`。

目标口径：

- 新增默认通用命令 `wx-auto-preview`。
- 调用微信开发者工具 CLI：`auto-preview`。
- 复用现有微信小程序目录选择、端口配置、端口缺失/不匹配可恢复重试链路。
- 不生成二维码、不回传图片；仅返回命令执行摘要与详情入口。

## 2. 现状证据

- 默认通用命令位于 `command_center/defaults.py`，已有 `wx-dev-preview` 与 `wx-dev-upload`。
- 微信命令目录选择与端口修复由 `bot.py` 中 `_is_wx_devtools_command`、`_maybe_handle_wx_preview`、`_apply_wx_preview_port_and_retry` 统一处理。
- `scripts/gen_preview.sh` 已具备小程序目录探测、端口映射读取、端口错误提示和微信 CLI `preview` 调用能力。
- 本机微信开发者工具 CLI 已确认支持 `auto-preview --project --info-output --port`。

## 3. 方案对比

| 方案 | 内容 | 优点 | 缺点 | 结论 |
|---|---|---|---|---|
| A | 新增 `wx-auto-preview`，复用现有微信目录/端口链路，脚本增加 `auto-preview` 模式 | 体验一致、复用端口修复、改动可控 | 需要改脚本/默认命令/bot/测试 | 采用 |
| B | 只新增裸命令直接调 CLI | 改动最少 | 不能复用目录选择和端口修复，失败体验差 | 不采用 |
| C | 新建完全独立脚本复制所有逻辑 | 隔离强 | 复制大量逻辑，维护成本高 | 不采用 |

1. 用户在命令管理点击或触发 `wx-auto-preview`。
2. bot 识别为微信开发者工具命令，扫描当前工作目录及一层子目录的小程序候选。
3. 用户选择小程序目录。
4. bot 注入 `PROJECT_PATH/PROJECT_BASE` 后执行默认命令。
5. `scripts/gen_preview.sh` 在 `WX_PREVIEW_ACTION=auto-preview` 下：
   - 解析小程序目录；
   - 解析 IDE HTTP 服务端口；
   - 调用 `cli auto-preview --project <path> --info-output <file> --port <port>`；
   - 成功时输出“手机自动预览已触发”，不输出 `TG_PHOTO_FILE`。
6. 若端口缺失或不匹配，沿现有可恢复链路提示输入/选择端口并重试。

## 9. 2026-05-31 迭代需求：单候选自动执行与端口失败自动重试

### 9.1 需求来源与理解

用户通过 Telegram 截图补充两个体验优化点：

1. `wx-dev-preview` 与 `wx-auto-preview` 在扫描范围内只有一个微信小程序目录时，不再要求用户点击目录选择，直接执行该目录。
2. 端口号失败时，先自动重试 1 次，再进入人工选择/输入端口流程。

本轮为 PLAN 只读调研与方案收敛，暂未改源码。

### 9.2 当前实现现状

| 现状 | 证据 |
|---|---|
| `wx-dev-preview`、`wx-auto-preview`、`wx-dev-upload` 都被 `_is_wx_devtools_command` 识别为微信开发者工具命令 | `bot.py`（锚点：`_is_wx_devtools_command`） |
| 命令执行前统一进入 `_maybe_handle_wx_preview` 扫描目录；只要有候选目录，就写入 FSM 并展示目录选择按钮 | `bot.py`（锚点：`_maybe_handle_wx_preview`、`_build_wx_preview_prompt`、`_build_wx_preview_keyboard`） |
| 目录选择确认后通过 `_wrap_wx_preview_command` 注入 `PROJECT_PATH/PROJECT_BASE`，再执行命令 | `bot.py`（锚点：`on_wx_preview_choice`、`_wrap_wx_preview_command`、`_execute_command_definition`） |
| 端口缺失或端口不匹配时，当前实现只进入人工端口选择/输入流程，不会自动重试 | `bot.py`（锚点：`_is_wx_preview_missing_port_error`、`_is_wx_preview_port_mismatch_error`、`WxPreviewStates.waiting_port`、`_apply_wx_preview_port_and_retry`） |
| `scripts/gen_preview.sh` 在 `preview/auto-preview` 两种模式下都会解析端口；失败时退出并让 bot 识别可恢复错误 | `scripts/gen_preview.sh`（锚点：`WX_PREVIEW_ACTION`、`_resolve_wx_devtools_port`、`未配置微信开发者工具 IDE 服务端口`、`auto-preview`） |
| 已有测试覆盖候选目录扫描、默认命令、脚本 auto-preview、端口解析/端口配置写入 | `tests/test_wx_preview_detection.py`；`tests/test_wx_preview_port_flow.py` |

### 9.3 方案对比

| 方案 | 内容 | 优点 | 风险/缺点 | 结论 |
|---|---|---|---|---|
| A. 推荐：高置信自动化 | 单候选自动执行；端口不匹配时优先使用 CLI 报错解析出的当前端口自动重试 1 次；缺失端口时仅在本地只识别到 1 个候选端口时自动重试，否则保留人工选择 | 用户点击最少；避免多 IDE/多端口误判；失败后仍有人工兜底 | 需要新增少量 helper 与异步测试 | 🌟 推荐 |
| B. 激进自动化 | 任何端口失败都拿第一个候选端口自动重试 | 改动更简单，自动化更强 | 多端口/多项目时容易写错端口并误触发 | 不推荐 |
| C. 只做单候选自动执行 | 不改端口失败流程 | 风险最低 | 未满足用户“端口失败自动重试一次”的明确诉求 | 不推荐 |

### 9.4 推荐开发设计

#### 9.4.1 单候选自动执行

受影响入口：`bot.py`（锚点：`_maybe_handle_wx_preview`）。

设计：

1. `_detect_wx_preview_candidates(base_dir)` 返回空：保持当前“未发现小程序目录”提示。
2. 返回 1 个候选：
   - 不写入 `WxPreviewStates.waiting_choice`；
   - 直接 `_wrap_wx_preview_command(command, only_candidate.project_root)`；
   - 复用 `_apply_command_env_overrides`；
   - 发送一条短提示，例如：`仅发现 1 个小程序目录，已自动选择：...`；
   - 调用 `_execute_command_definition(...)`。
3. 返回 2 个及以上候选：保持当前选择列表与取消按钮。
4. `wx-dev-upload` 暂不纳入本次自动跳过范围，除非用户明确要求；截图只点名两个 preview 命令，避免上传类命令误触发。

#### 9.4.2 端口失败自动重试 1 次

受影响入口：`bot.py`（锚点：`_execute_command_definition`、`_parse_wx_preview_port_mismatch`、`_suggest_wx_devtools_ports`、`_apply_wx_preview_port_and_retry`）。

推荐规则：

1. 仅对 `wx-dev-preview` 与 `wx-auto-preview` 生效，不扩展到 `wx-dev-upload`。
2. 若失败为端口不匹配，且 `_parse_wx_preview_port_mismatch(stderr_text)` 解析到 `mismatch_current_port`：
   - 自动把当前 IDE 端口写入 `wx_devtools_ports.json`；
   - 用 `PORT=<mismatch_current_port>` 自动重试 1 次；
   - 本次命令追加内部标记 `WX_DEVTOOLS_AUTO_PORT_RETRY=1`，避免无限递归。
3. 若失败为端口缺失，且 `_suggest_wx_devtools_ports()` 只返回 1 个端口：
   - 同样自动写入并重试 1 次。
4. 若没有端口、存在多个候选端口、自动重试后仍失败：保留当前人工端口选择/输入流程。
5. 自动重试前给 Telegram 一条短消息说明：`端口不匹配，已自动改用 IDE 当前端口 X 重试一次。若仍失败，会提示手动选择。`

### 10.5 风险与回滚

| 风险 | 状态 | 缓解 |
|---|---|---|
| 单候选自动执行减少用户确认 | 已控制 | 仅 preview/auto-preview 生效，upload 不自动跳过。 |
| 多端口环境误重试 | 已控制 | 端口缺失时只有唯一候选端口才自动重试；多候选回到人工流程。 |
| 自动重试递归 | 已控制 | `WX_DEVTOOLS_AUTO_PORT_RETRY=1` 标记防止第二次自动重试。 |
| 自动重试失败隐藏人工兜底 | 已控制 | 自动重试失败后仍进入现有 `waiting_port` 端口选择流程。 |

回滚方式：回退 `bot.py` 中 `_is_wx_preview_auto_flow_command`、单候选自动执行分支、端口自动重试 helper 与 `_execute_command_definition` 自动重试段；删除新增测试；恢复 `AGENTS.md` 微信开发命令约束行。
