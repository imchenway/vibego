# TASK_20260604_003 删除投递确认测试哨兵

## 1. 任务结论

本轮目标是彻底删除投递确认测试里的固定保留提示词，不再用“全局固定哨兵”替代真实自然语言测试输入。

最终实现口径：

1. `tests/test_plan_progress.py::test_session_ack_message_silent` 改为函数内局部随机普通文本。
2. 模拟 Codex JSONL 的 `input_text.text` 与 `DummyMessage.text` 共用同一个 `prompt_text`。
3. `bot.py::_dispatch_prompt_to_model` 增加 fail-closed 保护：整条 prompt 若是 vibego 内部测试标记样式，发送前阻断，不进入 tmux。
4. `docs/TASK_20260603_007_删除测试指令测试夹具.md` 已续写并移除固定哨兵沉淀。

## 2. 现状与根因

| 项目 | 当前实现/问题 | 证据 |
| --- | --- | --- |
| 投递确认测试 | 旧实现使用固定投递确认哨兵；当前已改为局部随机普通文本。 | `tests/test_plan_progress.py`（锚点：`prompt_text = f"hello from delivery confirmation test`、`test_session_ack_message_silent`） |
| 生产发送入口 | 所有 prompt 进入 tmux 前统一经过 `_dispatch_prompt_to_model`。 | `bot.py`（锚点：`_dispatch_prompt_to_model`、`dispatch_text = _prepend_enforced_agents_notice(prompt)`） |
| 新增保护点 | 纯内部测试标记样式的整条 prompt 在发送前阻断。 | `bot.py`（锚点：`INTERNAL_RESERVED_TEST_MARKER_RE`、`_is_internal_reserved_test_prompt`、`已阻断内部保留测试提示词进入 tmux`） |
| 回归测试 | 新增测试证明阻断发生在 tmux 前。 | `tests/test_task_description.py`（锚点：`test_dispatch_prompt_blocks_internal_reserved_vibego_test_marker_before_tmux`） |

根因：固定哨兵虽然便于测试断言，但会在仓库搜索、会话日志、排障语境里形成“像真实投递过的 prompt”的噪音。更优解是测试内用局部变量制造普通文本，同时在生产入口保留泛化的内部测试标记阻断。

## 7. 风险与回滚

| 风险 | 影响 | 缓解 |
| --- | --- | --- |
| 阻断规则过宽 | 用户发送类似内部标记时被误伤 | 仅 fullmatch 整条 prompt；普通需求描述包含类似片段不阻断。 |
| 测试随机文本不稳定 | JSONL 确认匹配失败 | `DummyMessage.text` 与 JSONL 统一引用 `prompt_text`，不做二次拼接。 |
| 全量测试既有失败掩盖新增失败 | 难以声明全量全绿 | 修改前已跑全量 baseline 并记录同三项失败；最终全量需对比失败是否未扩大。 |

回滚方式：

1. 删除 `bot.py` 中内部测试标记 regex、识别函数和 `_dispatch_prompt_to_model` 开头阻断块。
2. 删除 `tests/test_task_description.py` 新增阻断测试。
3. 保留或恢复 `tests/test_plan_progress.py` 的局部随机普通文本；不建议恢复固定保留提示词。
4. 回滚 docs/AGENTS 对应证据记录。
