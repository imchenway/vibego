## 6. 最小实现

### 6.1 长文本只保留附件消息

- `bot.py::reply_large_text(...)`
  - 长文本路径不再先发摘要消息
  - 改为直接发送：
    - `document`
    - `caption=摘要说明`
    - `reply_markup=attachment_reply_markup（或 reply_markup 兜底）`
  - 这样 Telegram 侧只保留一条附件消息

### 6.2 Codex 同时间戳双发收口

- `bot.py::_extract_codex_payload(...)`
  - 为 `response_item.message / assistant_message` 增加内部 metadata 标记
- `bot.py::_collapse_codex_response_item_duplicates(...)`（新增）
  - 当同一时间戳同时出现：
    - `assistant_message`
    - `message`
  - 仅保留正式 `message`
- `bot.py::_read_session_events_jsonl(...)`
  - 在读完一批事件后统一执行上述收口

### 6.3 原生按钮布局

- `bot.py::_build_model_quick_reply_keyboard(...)`
  - 原生消息改为：

```text
[✅ 全部按推荐] [🧩 部分按推荐（需补充）]
[⬆️ 提交分支] [🧪 任务状态更新为测试中]
```

### 6.4 并行按钮布局

- `bot.py::_build_model_quick_reply_keyboard(...)`
  - 并行消息改为：

```text
[✅ 全部按推荐] [🧩 部分按推荐（需补充）]
[⬆️ 提交并行分支] [🧪 任务状态更新为测试中]
[🏷 /TASK_xxxx 标题] [↩️ 回复 /TASK_xxxx]
```
