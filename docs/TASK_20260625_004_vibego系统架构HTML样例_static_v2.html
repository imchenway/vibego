控制链路

vibego CLI / Master Bot 通过 master.py 读取项目配置，再调用 run_bot.sh 或 stop_bot.sh 管理 worker。

业务链路

Project Worker Bot 接收消息，bot.py 完成入站规整，经 Dispatch gate 投递到 tmux 模型 CLI。

会话回传

模型 CLI 写入 Session JSONL，binder 绑定 pointer，watcher 读取事件后回传 Telegram。

数据边界

项目配置、任务命令数据、日志、状态默认写入 ~/.config/vibego，避免污染仓库。

如何阅读此图

默认说明

先看“主链路 8 步”，再看“分层模块地图”。如果 Telegram 不执行 JS，左侧主内容仍完整可读；如果浏览器执行 JS，点击节点会更新这里的详情。

质量自检

  • 主内容静态写入 DOM。
  • JS 只做交互增强，不负责首屏渲染。
  • 跨层关系用关系表表达，避免路径混乱。
  • PNG fallback 可通过浏览器截图或打印生成。
