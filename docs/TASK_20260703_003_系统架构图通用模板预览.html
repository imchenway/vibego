系统架构模板：可变槽位版预览

本版不再固定 emoji、业务文案或“Agent / RAG / LLM”类结构；只固定视觉语法：边界、分区、箭头、图例和信息密度。每个节点的图标、名称、职责、分组名称都由数据槽决定，可按系统类型隐藏或替换。

模板原则：固定的是视觉语法，不是架构答案。每个区域都可以改名、隐藏、拆分或合并；图标使用 icon 槽，由生成时按实际系统选择 emoji、SVG 小图标、字母徽标或留空。

可变槽位契约
layout.profile可选：业务应用、数据平台、基础设施、AI 系统、集成网关、中台/平台。profile 只影响命名和默认区域，不改变视觉质量门禁。
regions[]每个区域有 title、visible、role、nodes[]；入口 rail、侧栏、支撑层、底座都可隐藏。
nodes[]节点只固定 icon/title/subtitle/tone/detail 槽；图标不是预设业务语义，可用 emoji、SVG、缩写或空。
flows[]只要求源、目标、类型和标签；不预设必须是“Agent/RAG/LLM”链路。
naming“核心应用区”“支撑层”“数据层”只是默认显示名，可替换为“控制平面 / 数据平面 / 执行平面 / 运行环境”等。
constraints禁止接口流水线标题、编号实现分层、源码路径堆主图；允许按系统事实重命名与删减区域。
optional虚线节点和 opt 图标表示可选槽，不代表具体业务能力。
quality仍需保留宽画布、清晰边界、节点居中、箭头有源有目标、无横向滚动。
