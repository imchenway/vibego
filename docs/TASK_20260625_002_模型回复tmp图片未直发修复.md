# TASK_20260625_002 模型回复 /tmp 图片未直发修复

## 1. 背景与目标

- 现象：模型最终回复中已经给出本地图片产物路径，例如 `/tmp/fawn-platform-admin-B-refinements.png`，Telegram 只展示文本路径，没有补发可预览图片。
- 目标：保持项目内图片直发能力不变，同时允许模型显式引用的近期 `/tmp`/`TMPDIR` 图片在安全校验通过后直发到 Telegram。
- 非目标：不放开任意项目外文件回传；不改变 Telegram 文本投递、Markdown 表格渲染、PlanConfirm、任务绑定等其它消息流程。

## 2. 证据链

| 证据 | 结论 |
|---|---|
| `bot.py`（锚点：`_collect_model_response_local_images`、`_resolve_model_response_local_image_path`） | 旧逻辑只收集项目根目录内图片；绝对路径若不在 `roots` 内会被跳过。 |
| `docs/TASK_20260624_003_Telegram模型回复本地图片直接预览.md`（锚点：`需求边界`） | 旧契约明确“项目目录外绝对路径不发”，因此 `/tmp/*.png` 被排除是符合旧实现但不符合当前用户场景。 |
| 用户截图与会话 JSONL（`/Users/david/.codex/sessions/2026/06/24/rollout-2026-06-24T17-42-59-019ef902-ee6e-77a0-a0a7-7f25352955f0.jsonl`，锚点：final answer 文本包含 `/tmp/fawn-platform-admin-B-refinements.png`） | 模型确实在最终回复中显式引用了 `/tmp` 图片路径，Telegram 展示层没有补发图片。 |
| `~/.config/vibego/logs/codex/fawnstudio/run_bot.log`（锚点：2026-06-25 10:08 左右“模型输出发送成功”） | 当时只有文本发送成功日志，没有“模型输出图片发送成功”日志。 |

## 3. 根因

旧安全边界把“项目目录外路径”统一视为不可发送，避免误发本机文件。该边界对任意路径是正确的，但没有覆盖模型常用的临时图片产物目录 `/tmp`。因此模型生成图后只要产物在 `/tmp`，即使最终回复明确引用，也会被 `_resolve_model_response_local_image_path` 拦截。

## 4. 契约变更

### 4.1 新行为

1. 模型最终回复先发送文本，再补发图片的顺序不变。
2. 项目根目录内 PNG/JPEG/WebP 图片直发规则不变。
3. 项目外路径默认仍不发送。
4. 仅当模型文本显式引用的是绝对 `/tmp`、`/private/tmp` 或 `TMPDIR` 下图片，并同时满足以下条件时才直发：
   - 扩展名属于 `.png/.jpg/.jpeg/.webp`；
   - 文件存在且是普通文件；
   - MIME 推断为空或以 `image/` 开头；
   - 文件大小不超过 `TELEGRAM_MODEL_TMP_IMAGE_MAX_BYTES`（默认 20MB）；
   - 文件 mtime 与模型事件 timestamp 的差值不超过 `TELEGRAM_MODEL_TMP_IMAGE_MAX_AGE_SECONDS`（默认 1800 秒）。
5. `/tmp` 图片直发失败时仍复用已有降级逻辑：先尝试 `send_photo`，失败后 `send_document`。

### 4.2 不变边界

| 边界 | 处理 |
|---|---|
| 数据库 | 不涉及表结构、迁移、运行期 DB。 |
| 对外 API | 不涉及 HTTP/开放接口。 |
| Telegram 入站消息 | 不改用户消息、附件下载、相册聚合逻辑。 |
| 任务/并行会话 | 不改任务绑定、并行 session、PlanConfirm。 |
| 构建/依赖 | 不新增依赖，不改构建链。 |

## 8. 风险与回滚

| 风险 | 控制 |
|---|---|
| 误发历史 `/tmp` 文件 | 必须 mtime 与模型事件 timestamp 在默认 30 分钟窗口内。 |
| 误发大文件 | 默认 20MB 上限，超过不发送。 |
| macOS `/tmp` 实际解析为 `/private/tmp` | 同时收集 `/tmp`、`/private/tmp`、`TMPDIR` 并用 resolved path 判断。 |
| 误发非图片 | 扩展名与 MIME 双重过滤。 |
| 线上需回滚 | 删除 `/tmp` 安全白名单逻辑，恢复旧的“仅项目根目录内图片可直发”契约；无需 DB 回滚。 |
