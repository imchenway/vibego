# /TASK_0082 任务详情点击已完成，响应太慢了

## 1. 背景

- 用户反馈：任务详情页点击 `✅ 已完成` 后，响应太慢。
- 本次已确认口径：
  - 慢点：**刷新慢**
  - 期望：**留在详情页**
  - 时延目标：**1 秒内有明确反馈**
  - 修复范围：**仅“已完成”**
  - 最终决策：**只要状态修改成功就立即返回，后续清理流程交给异步**

## 2. 证据

- 任务状态按钮入口：`bot.py`（锚点：`_build_status_buttons`, `_build_task_actions`）
- 状态点击处理：`bot.py`（锚点：`on_status_callback`）
- 详情重渲染链路：`bot.py`（锚点：`_render_task_detail`, `_load_task_context`）
- 详情正文不展示状态字段：`bot.py`（锚点：`_format_task_detail`）
- 并行清理链路：`bot.py`（锚点：`_cleanup_parallel_session_workspace_safely`）

## 5. TDD 红灯

先修改测试，要求：

1. `done` 写库成功后应**立即返回**
2. `done` 的详情按钮刷新与并行清理应进入**同一个后台收尾任务**
3. `done` 快路径不应再同步调用 `_render_task_detail` / `_try_edit_message`

首次执行：

```bash
python3.11 -m pytest -q tests/test_task_description.py -k "on_status_callback_done_schedules_parallel_cleanup or on_status_callback_non_done_does_not_schedule_parallel_cleanup"
```

结果：

- ❌ `test_on_status_callback_done_schedules_parallel_cleanup`
- 失败原因：当前实现仍会同步调用 `_render_task_detail(updated.id)`，未满足“立即返回 + 后台收尾”

## 6. 最小实现

### 6.1 done 快路径立即返回

- `bot.py::on_status_callback`
  - 当 `updated.status == "done"` 时：
    - 立即 `asyncio.create_task(_finalize_done_status_update_safely(...))`
    - 立即调用 `_answer_callback_safely(callback, "状态已更新")`
    - 主流程直接 `return`

### 6.2 后台收尾拆分

- `bot.py::_refresh_done_task_detail_markup_safely(...)`
  - 后台仅刷新任务详情按钮
  - 不同步重渲染整段详情文本
- `bot.py::_finalize_done_status_update_safely(...)`
  - 先后台刷新按钮
  - 再后台执行并行运行态清理

### 6.3 边界处理

- 后台刷新按钮失败：
  - 仅记录日志
  - 不影响前台已经返回的成功提示
- `message is None` 时：
  - 仅跳过按钮刷新
  - 仍可继续后台清理
