# TASK_20260303_001 项目列表点击启动后状态未变更修复

## 背景
用户反馈：Telegram 项目列表点击“启动”后状态没有变更；但 `tmux ls` 可看到会话已创建。

## 结论
不是按钮回调失效，而是 **worker 启动后立即崩溃**，导致 master 健康检查失败并回写 `stopped`。

## 关键证据
- `master.py:2120-2131`：健康检查失败时执行 `status="stopped"`，成功才写 `status="running"`。
- `master.py:1956-2020`：健康检查依赖日志中出现 `Telegram 连接正常`。
- `scripts/run_bot.sh:339-342`：先创建 tmux 会话，再 `exec bot.py`，会出现“tmux 在、worker 挂”的现象。
- `~/.config/vibego/logs/codex/vibegobot/run_bot.log`：
  - `SyntaxError: unterminated string literal (detected at line 350)`
- `~/.config/vibego/state/master_state.json`：`vibegobot.status` 为 `stopped`。

## 修复内容
1. 修复 `bot.py` 中 `ENFORCED_AGENTS_NOTICE` 字符串未闭合问题（改为括号拼接 + 显式换行）。
2. 为立即恢复线上运行，同步修复 pipx 安装副本：
   - `~/.local/pipx/venvs/vibego/lib/python3.11/site-packages/bot.py`
   - 并生成备份：`bot.py.bak.20260303_192416`
