# TASK_20260630_017 vibe-diagram 业务架构图领域地图二次优化

## 优化目标

在 TASK_016 的基础上继续减少“卡片感”，强化图形关系：对象生命周期主轴、业务域扇出、规则覆盖条和反馈闭环。

## 主要变化

- 从五层同权网格改为一条对象生命周期主轴：用户 → AppConfig → Session/Message → Scene/Slot → AnswerSource → 裁决 → 输出。

- 把身份、真实数据、财务、转人工、回滚做成覆盖条，显示规则作用在哪些对象/阶段上。
- 保留点击详情承载证据，避免主图底部证据堆叠。
