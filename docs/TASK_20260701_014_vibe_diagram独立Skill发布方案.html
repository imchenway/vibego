可见性与更新：别人能不能搜到、以后怎么同步

结论先行

别人现在能搜到吗？

不能理解成“全网/所有 Codex 用户都能搜到”。当前实现是把 vibe-diagram 发布到本机 native skill 目录：/Users/david/.codex/skills/vibe-diagram 与 /Users/david/.agents/skills/vibe-diagram。这意味着本机新的 Codex/agent 会话可按 native skill registry 发现；其他人的机器只有在安装/更新 vibego 并执行同步后才会看到。

如果要“别人搜索可安装”

还需要单独做发布渠道：例如把 vibe-diagram 做成可安装的 Codex skill / plugin 包，并发布到对应技能源或插件市场。本轮交付只完成本机 native 发布与 vibego 同步链路，不等同于公开市场发布。

以后怎么更新 skill 内容

改仓库事实源 vibego_cli/data/skills/vibe-diagram/SKILL.md 或 references/*.md，跑相关测试，然后执行 python3.11 -m vibego_cli agents-sync --source-root /Users/david/hypha/tools/vibego --json。同步会把新版本复制到 override 目录和 native skill 目录。

其他用户/worker 怎么拿到更新

其他用户需要先更新 vibego 仓库或安装包，再运行 agents-sync 或重启 worker 触发 sync_agents_block()。已经运行很久的 worker 可能有旧上下文；需要立即生效时重启对应 worker。
