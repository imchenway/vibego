# /TASK_20260312_003 已处于测试状态也自动展示任务列表

## 1. 背景

用户反馈：

- 点击模型答案消息底部的 `🧪 任务状态更新为测试中` 按钮后，
- 当任务已经处于 `test` 状态时，当前只会提示：

```text
任务已处于“测试”状态
```

- 但**不会继续展示任务列表**，与“非 test -> test”分支的体验不一致。

附件截图证据：

- `/Users/david/.config/vibego/data/telegram/vibegobot/2026-03-12/20260312_115112202-b6f79fb11485.jpg`

## 2. 现状证据

### 2.1 非 test -> test 分支

`bot.py:on_model_task_to_test`

- 任务状态更新成功后，会执行：

```python
await _handle_task_list_request(callback.message)
```

### 2.2 已是 test 分支

同函数中原逻辑：

```python
if task.status == "test":
    await callback.answer("任务已处于“测试”状态")
    return
```

因此不会展示任务列表。

### 2.3 既有测试资产

`tests/test_task_description.py`

- 已覆盖：
  - 正常切到测试后展示任务列表
  - 已是测试
  - 任务不存在
  - 任务 ID 无效

## 3. 方案

按推荐方案，仅修改 **模型答案消息按钮** 这条链路：

- 当任务已是 `test` 时：
  1. 继续 toast：`任务已处于“测试”状态`
  2. 若 `callback.message` 存在，则继续调用 `_handle_task_list_request(callback.message)`

不扩散到：

- 任务详情页状态按钮
- 其它状态流转按钮
