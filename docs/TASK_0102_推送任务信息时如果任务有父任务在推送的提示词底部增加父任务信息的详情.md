## 6. 最小实现

### 6.1 父任务信息区块
- 在 `bot.py` 新增：
  - `_build_task_prompt_related_task_code()`
  - `_build_task_prompt_attachment_lines()`
  - `_build_parent_task_prompt_lines()`
- 目的：
  - 统一“关联任务编码”口径
  - 统一“其他附件（未归属步骤）”拼装
  - 统一父任务详情区块的输出格式

### 6.2 推送链路自动加载父任务详情
- 新增 `bot._load_parent_task_context_for_model()`
- 行为：
  - 按 `related_task_id` 读取父任务详情
  - 读取父任务附件并转为发送顺序
  - 任一步失败时回退旧历史区块，避免影响主链路

### 6.3 提示词底部完全替换
- 修改 `bot._build_model_push_payload()`
- 修改 `bot._build_task_context_block_for_model()`
- 行为：
  - 若能拿到父任务详情，则底部改为：
    - `以下为父任务信息，用于辅助回溯任务处理记录：`
    - 父任务标题 / 编码 / 结构化描述 / 补充任务描述 / 关联任务编码 / 其他附件
  - 否则保持原：
    - `以下为任务执行记录，用于辅助回溯任务处理记录：`

## 8. 兼容性与风险

### 8.1 兼容性
- 仅在能成功读取 `related_task_id` 对应任务时替换底部区块
- 若无关联任务或关联任务不存在，保持旧“任务执行记录”逻辑
- `done -> /compact` 语义不变

### 8.2 风险点
- 当前“父任务”采用 `related_task_id` 业务口径，而非真实 `parent_id`
- 该口径与仓库现状一致，但后续若重新启用真实子任务体系，需要重新审视命名与展示文案
