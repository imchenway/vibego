## 6. 最小实现

### 6.1 缺陷结构升级为四字段
- 新增字段常量：
  - `bot.py`（锚点：`DEFECT_CURRENT_STATE_LABEL`、`DEFECT_EXPECTED_EFFECT_LABEL`）
- 新增四字段解析：
  - `bot.py`（锚点：`_parse_defect_description_fields`）
- 新增四字段构建：
  - `bot.py`（锚点：`_build_defect_description`）

### 6.2 新旧缺陷格式并存
- 新缺陷统一输出：`前置条件 / 复现步骤 / 现状 / 预期效果`
- 历史三字段缺陷保持：`前置条件 / 复现步骤 / 预期结果`
- 更老双字段缺陷继续沿用当前兼容逻辑：补 `前置条件：-`
- 实现位置：
  - `bot.py`（锚点：`_get_structured_task_fields`、`_format_task_detail`、`_append_task_prompt_description_fields`）

### 6.3 交互式创建流程改造
- `/task_new` 缺陷创建：
  - `复现步骤 -> 现状 -> 预期效果`
  - `tasks/fsm.py`（锚点：`TaskCreateStates.waiting_current_state`、`TaskCreateStates.waiting_defect_expected_effect`）
  - `bot.py`（锚点：`on_task_create_current_state`、`on_task_create_defect_expected_effect`）
- `报告缺陷` 流程：
  - `复现步骤 -> 现状 -> 预期效果`
  - `tasks/fsm.py`（锚点：`TaskDefectReportStates.waiting_current_state`、`TaskDefectReportStates.waiting_expected_effect`）
  - `bot.py`（锚点：`on_task_defect_report_current_state`、`on_task_defect_report_expected_effect`）
- 确认阶段继续补充文本/附件时，统一追加到最后一个字段 `预期效果`
  - `bot.py`（锚点：`补充{DEFECT_EXPECTED_EFFECT_LABEL}`）

### 6.4 命令式创建与帮助文案
- `/task_new ... | type=缺陷` 新推荐参数：
  - `precondition`
  - `reproduction`
  - `current_state`
  - `expected_effect`
- 旧参数 `expected_result` 继续兼容，但按**新四字段**创建
- 实现位置：
  - `bot.py`（锚点：`_prepare_task_new_command_description`、`on_tasks_help`）

### 6.5 Excel 模板与导入
- 新模板表头：
  - `缺陷标题 / 前置条件 / 复现步骤 / 现状 / 预期效果 / 关联任务编码 / 优先级`
- 兼容旧模板表头：
  - `缺陷标题 / 前置条件 / 复现步骤 / 预期结果 / 关联任务编码 / 优先级`
- 旧模板导入时：
  - `现状` 默认为空
  - `预期结果` 映射到新字段 `预期效果`
- 实现位置：
  - `bot.py`（锚点：`DEFECT_EXCEL_TEMPLATE_HEADERS`、`DEFECT_EXCEL_TEMPLATE_LEGACY_HEADERS`、`_parse_defect_excel_import_file`）
