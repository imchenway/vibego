# /TASK_0101 附件上传后的路径展示位置

## 1. 任务背景

- 任务标题：附件上传后的路径展示位置
- 任务编码：/TASK_0101
- 用户目标：创建任务/缺陷时，任一步骤上传的附件都应显示在对应步骤位置，而不是统一塞到底部附件列表。
- 已确认决策：
  1. 生效范围：全部创建入口
  2. 展示方式：仅步骤内展示
  3. 确认页补附件：归到最后一步

## 9. 兼容性与风险

### 9.1 兼容策略
- 不改数据库 schema，复用现有 `description + task_attachments` 语义。
- 历史老任务若没有 `[附件:...]` 内联语义，仍会在底部“其他附件”看到附件。
- `/attach` 后续追加的附件未强行绑定到业务步骤，仍展示为“其他附件”。

### 9.2 风险点
- 步骤附件归属依赖 `[附件:路径]` 文本解析；若历史数据里手工写入相同格式，会被视为已归属附件。
- 该风险目前可接受，因为展示逻辑本身就是围绕仓库内现有 `[附件:...]` 表达式展开。
