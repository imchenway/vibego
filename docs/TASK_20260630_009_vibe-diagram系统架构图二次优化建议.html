  • 缺口更像状态，而不是解释文字。

不要为了“更完整”再铺细节

系统架构图只回答“系统由哪些层构成、请求如何走、数据在哪里、兜底在哪里”。实现调用链应交给代码时序图。

  • 不展开所有 handler。
  • 不把源码路径放回主画布。
  • 不把异常分支画成蜘蛛网。

Layer 05数据与观测PostgreSQL / pgvector / Trace

E7Agent 编排

识别场景、补槽、执行、追问；handler registry 保持为扩展点。

E13缺口角标PostgreSQL / pgvector / Trace

数据面和观测面收束到南侧，RAG/Adapter 缺口用角标标明。

Fallback Rail七鱼人工兜底

从入口到 Agent 都可旁路进入，视觉上应是右侧边界，不是数据面节点。

触发条件低置信 / 自助关闭 / 工具不可用

触发条件放在 rail 内，避免在主链路到处拉虚线。

终点人工服务闭环

终点仍可见，但不再和 PostgreSQL / Trace 同层竞争注意力。

同步语义

主链路 HTTP REST；无独立 MQ。

控制语义

Feature Flags + Guard 作为侧挂策略。

缺口语义

Adapter / RAG 待接入用红色角标。

追踪语义
