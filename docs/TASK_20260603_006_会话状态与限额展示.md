## 3. 需求口径

### 3.1 用户/场景

- 使用者：通过 Telegram 操作 vibego worker 的用户。
- 使用时刻：想确认当前主 Codex 会话模型、推理等级、上下文窗口和限额占用时。
- 业务目标：减少进入终端或翻 JSONL 的成本，快速判断是否接近 5h/周限额。

### 3.2 核心流程

1. 用户在 Telegram 发送 `/status`。
2. Worker 读取 `CODEX_SESSION_FILE_PATH` pointer。
3. 解析 pointer 指向的 Codex JSONL。
4. 从最新结构化事件提取：
   - 模型：优先 `turn_context.payload.collaboration_mode.settings.model`，兜底 `payload.model` 与 `ACTIVE_MODEL`。
   - 推理等级：`turn_context.payload.collaboration_mode.settings.reasoning_effort`。
   - Context：优先最新 `token_count.info.model_context_window`，兜底 `task_started.model_context_window`；若最新 `token_count.info.last_token_usage.total_tokens` 存在，则按 Codex CLI 口径计算剩余百分比。
   - 限额：最新 `token_count.rate_limits` 中 `window_minutes=300` 作为 5h，`window_minutes=10080` 作为周限额；展示 `100 - used_percent` 的剩余百分比，并在超过边界时 clamp 到 `0%~100%`。
5. Worker 向 Telegram 回一条 Markdown 文本。

### 3.3 边界与异常

| 场景 | 处理 |
| --- | --- |
| `CODEX_SESSION_FILE_PATH` 未配置、pointer 缺失、pointer 指向文件不存在 | 显示“未绑定当前会话” |
| JSONL 中没有 `token_count` | 显示“暂无限额数据，等待下一次模型事件” |
| JSONL 坏行/不可读字段 | 跳过坏行或显示“未知”，不阻断 `/status` |
| 多条 `token_count` | 使用最新一条 |
| 最新 `token_count` 缺少 `last_token_usage` | Context 剩余显示“未知”，但仍展示窗口大小与限额 |
| 非 Codex 或字段缺失 | 对应字段显示“未知/暂不支持”口径中的“未知” |

## 5. 契约变更

### 5.1 Telegram 命令

```text
/status — 查看模型与限额状态
```

### 5.2 输出格式

```text
*会话状态*
模型：gpt-5.5
推理等级：xhigh
Context：94% 剩余（窗口 243,200 tokens）
限额：5h 剩余 94%（20:43 重置） · 周 剩余 35%（06-08 07:41 重置）
```

### 5.3 安全约束

- `/status` 只读本地 JSONL。
- 禁止向 tmux/Codex 发送输入。
- 禁止调用外部 API 获取限额。
- 解析失败 fail-soft，不影响普通消息推送、`/session_live`、`/goal`。

### 5.4 Context 剩余百分比口径

按 Codex CLI 当前源码口径对齐：

```text
BASELINE_TOKENS = 12000
effective_window = model_context_window - BASELINE_TOKENS
used = max(last_token_usage.total_tokens - BASELINE_TOKENS, 0)
remaining_percent = round(clamp((effective_window - used) / effective_window * 100, 0, 100))
```

证据：

- Codex 官方手册：`/status` 用于查看 thread ID、context usage 与 rate limits。
- openai/codex 源码：`chatwidget.rs` 通过 `last_token_usage.percent_of_context_window_remaining(window)` 更新 context；`token_usage.rs` 中 `BASELINE_TOKENS = 12000` 且按 `last_token_usage.total_tokens` 计算剩余百分比。

不采用口径：

- 不用 `total_token_usage.total_tokens`：该值是累计会话总量，可能远超 context window，不能表示当前窗口剩余。
- 不只用 `input_tokens`：用户明确要求与 Codex CLI `/status` 一样。

## 9. 风险与回滚

| 风险 | 影响 | 缓解 |
| --- | --- | --- |
| Codex JSONL 字段未来变化 | `/status` 显示未知或暂无限额数据 | 解析函数跳过坏行、字段缺失 fail-soft；后续只需调整解析测试与函数 |
| pointer 未绑定 | 用户误以为命令不可用 | 明确显示“未绑定当前会话” |
| Telegram Markdown 特殊字符 | 状态消息发送失败 | 复用 `_answer_with_markdown` 的 Markdown guard |

回滚方式：删除 `bot.py` 中 `/status` 命令、handler 与解析函数，删除新增测试与 AGENTS/doc 证据；因无 DB/配置变更，无数据回滚。
