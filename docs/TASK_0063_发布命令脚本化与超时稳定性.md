# TASK_0063：发布命令脚本化与超时稳定性

## 背景
- 现象：按钮执行 `pipx-publish-patch` 经常在 120 秒超时。
- 对比：同一命令在本地终端可成功执行。
- 目标：将发布流程脚本化，降低“注入命令过长 + 非交互环境差异”导致的不稳定。

## 本次改动
- 重构 `scripts/publish.sh`
  - 去除硬编码项目路径，改为脚本自定位仓库根目录。
  - 新增非交互认证优先级：
    - `TWINE_USERNAME` + `TWINE_PASSWORD`
    - `PYPI_API_TOKEN`（自动映射 `__token__`）
    - keyring（仅作为兜底）
  - 默认使用 `twine upload --non-interactive`，避免机器人环境卡住输入。
  - 增加可选参数：
    - `--skip-upload`
    - `--skip-pipx`
    - `--skip-restart`
    - `--dry-run`
- 新增 `scripts/publish_patch.sh`
  - 作为 patch 发布快捷入口，便于命令中心直接调用。

## 推荐命令中心配置
- 命令：
  - `bash "$ROOT_DIR/scripts/publish_patch.sh"`
- 建议超时：
  - 至少 `900` 秒（发布链路包含构建、上传、pipx 重装，120 秒过短）。
