# TASK_20260310_009 任务完成后异步清理并行运行态

## 1. 背景

用户明确要求：

- 当任务点击 `done` 后
- 立即异步清理该任务相关的：
  - 并行 tmux
  - 并行 state / watcher / callback binding
  - `~/.config/vibego/runtime/parallel/<project>/<TASK_ID>` 目录

不保留未提交改动保护，也不增加额外确认。

## 5. 最小实现

### 5.1 新增后台清理调度

- `_cleanup_parallel_session_workspace_safely(task_id)`
  - 后台调用 `_delete_parallel_session_workspace(...)`
  - 捕获异常并写日志，避免影响前台状态更新
- `_schedule_parallel_cleanup_for_done(task_id)`
  - 仅负责 `asyncio.create_task(...)` 调度

### 5.2 任务状态更新后立即异步触发

- `on_status_callback(...)`
  - `TASK_SERVICE.update_task(...)` 成功后
  - 若 `updated.status == "done"`
    - 立即调用 `_schedule_parallel_cleanup_for_done(updated.id)`

### 5.3 保持幂等

- 若该任务没有并行会话记录：
  - 清理函数静默返回
- 若 tmux/目录已不存在：
  - 继续收尾，不影响用户看到的“状态已更新”
