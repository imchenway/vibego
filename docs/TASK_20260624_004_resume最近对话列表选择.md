# TASK_20260624_004 Resume 最近对话列表选择

## 1. 背景与目标

用户反馈：当前 `resume` 功能需要用户手动复制并输入 `sessionId`，但实际使用时更希望先看到“该项目最近的对话列表”，点击其中一个后直接 resume。

目标：在 Telegram `会话实况 -> Resume 会话` 流程中，先展示当前项目最近可恢复的 Codex 会话列表；用户点击列表项后，vibego 自动重启主 Codex CLI 并绑定该会话。手动输入 `sessionId` 作为二级兜底入口保留。

## 2. 现状与根因

现状证据：

- `bot.py::on_session_live_resume_callback` 原流程进入 `SessionResumeStates.waiting_session_id`，提示用户手动输入 `sessionId`。
- `bot.py::_resume_main_session_from_user_input` 已具备按当前项目 cwd 校验 session、重启主 tmux、绑定 watcher 的恢复能力。
- `bot.py::_resolve_main_session_binding_target` 已按 JSONL 文件 stem / Codex UUID 查找 session，并通过 `_read_session_meta_cwd` 与 `_paths_equal` 拒绝非当前项目 session。

根因：已有恢复能力只提供“手输 id”入口，没有把当前项目的历史 JSONL 会话扫描结果转成 Telegram 可点击列表，导致移动端操作成本高且容易复制错误。

## 4. 契约变更

- `会话实况 -> ↩️ Resume 会话` 不再直接要求手动输入 `sessionId`，而是先展示“当前项目最近可恢复会话”列表。
- 最近会话来源：扫描主会话搜索根目录下匹配 `MODEL_SESSION_GLOB`（默认 `rollout-*.jsonl`）的 Codex JSONL 文件。
- 安全边界：
  - 只列出 JSONL `cwd` 与当前 `MODEL_WORKDIR` / `CODEX_WORKDIR` / 项目主工作目录一致的会话；
  - 缺失 cwd 或 cwd 不匹配的会话不展示，避免跨项目串会话；
  - Telegram callback 只携带短 token，不直接携带本地路径或完整 sessionId；
  - 短 token 默认 15 分钟过期，过期后提示刷新最近会话。
- 列表排序：按 JSONL 文件 mtime 倒序，默认最多展示 `SESSION_RESUME_RECENT_LIMIT=8` 个。
- 点击列表项后：复用既有主会话恢复逻辑，重启主 tmux 并绑定为当前 Telegram 主会话。
- 兜底入口：最近会话页保留“手动输入 sessionId”，继续支持文件 stem、Codex UUID 或 `sessionId : xxx` 行。
- 非 Codex 模型：继续 fail-closed，提示暂仅支持 Codex resume。

## 5. 实现摘要

1. 新增 `SESSION_LIVE_RESUME_SELECT_PREFIX`、刷新与手动 callback 常量。
2. 新增 `MainSessionResumeRecentBinding` 与 `SESSION_RESUME_RECENT_BINDINGS`，用于将 Telegram 短 token 映射到已校验的 `MainSessionBindingTarget`。
3. 新增 `_list_recent_main_session_binding_targets()`：扫描候选 JSONL、按当前项目 cwd 过滤、mtime 倒序截断。
4. 新增 `_build_session_resume_recent_view()` 与 `_show_session_resume_recent_view()`：构造 Telegram 最近会话选择页。
5. 新增 `on_session_live_resume_refresh_callback`、`on_session_live_resume_manual_callback`、`on_session_live_resume_select_callback`。
6. 抽出 `_resume_main_session_to_target()`，让“手输 sessionId”和“点击最近会话”复用同一恢复/绑定逻辑。

## 8. 风险与回滚

| 风险 | 缓解 |
| --- | --- |
| 跨项目 session 被误展示 | 必须读取 JSONL meta cwd，并与当前项目工作目录一致才展示。 |
| Telegram callback data 过长或泄露本地路径 | callback 只放 12 位短 token，本地路径保存在进程内存映射中。 |
| token 过期后点击失败 | 给出“请刷新最近会话列表”的可见提示。 |
| 会话根目录过大导致扫描变慢 | 默认只取 mtime 倒序前 8 个；后续若真实慢，可改为优先读取 `recent_sessions.json` 并保留扫描兜底。 |
| 用户仍需要恢复更老会话 | 保留“手动输入 sessionId”入口。 |

回滚方式：移除 `bot.py` 中最近会话常量、`MainSessionResumeRecentBinding`、最近会话扫描/视图/回调函数，并将 `on_session_live_resume_callback` 恢复为直接进入 `SessionResumeStates.waiting_session_id`；删除新增测试与 AGENTS 事实行。
