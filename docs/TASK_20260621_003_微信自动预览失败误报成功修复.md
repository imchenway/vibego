# TASK_20260621_003 微信自动预览失败误报成功修复

## 1. 背景与目标

用户反馈：微信自动预览显示成功，但随后生成二维码明确失败；既然二维码/预览编译失败，自动预览不应显示成功。

目标：`wx-auto-preview` 不能只依据 `auto-preview` CLI 退出码判断成功，必须先通过与 `wx-dev-preview` 同源的 preview 编译与二维码产物校验；校验失败时，Telegram 必须展示失败态和诊断摘要。

## 2. 现状与根因

运行证据：

- `master_commands.db` 历史 `id=754`：`wx-auto-preview`，`status=success`，`exit_code=0`，输出为“手机自动预览已触发”。
- `master_commands.db` 历史 `id=755`：`wx-dev-preview`，`status=failed`，`exit_code=3`，错误为 `CLI 未生成二维码文件`，根因包含 WXML 编译错误 `Bad attr wx:elif ... unexpected token .`。

代码根因：

- `scripts/gen_preview.sh` 的 `WX_PREVIEW_ACTION=auto-preview` 分支只调用 `cli auto-preview --info-output --port`，没有执行 preview 编译校验，也没有校验二维码产物。
- `bot.py::_execute_command_definition` 已按 `exit_code == 0` 判断成功；误报来自脚本把“auto-preview 命令触发成功”当成“预览链路成功”。

## 3. 受影响范围

| 范围 | 是否影响 | 说明 |
| --- | --- | --- |
| `scripts/gen_preview.sh` | 是 | auto-preview 前新增 preview 编译与二维码产物校验。 |
| `tests/test_wx_preview_port_flow.py` | 是 | 覆盖编译失败、未生成二维码、成功但不回传二维码三类脚本行为。 |
| `tests/test_command_execution_flow.py` | 是 | 覆盖 auto-preview 非 0 时 Telegram 显示失败态与诊断。 |
| `bot.py` | 否 | 当前执行结果判定已基于 exit code；本次不需要改。 |
| 数据库/API/依赖 | 否 | 无 schema、API、依赖变化。 |

## 4. 契约变更

- `wx-auto-preview` 成功必须满足两步：
  1. 内部 `cli preview` 退出码为 0；
  2. 内部校验二维码文件真实存在；
  3. 随后 `cli auto-preview` 退出码为 0。
- 内部校验二维码只用于证明编译/preview 链路成功，不输出 `TG_PHOTO_FILE`，不回传 Telegram 图片。
- 成功文案改为“编译校验通过，手机自动预览已触发”，避免暗示手机端一定已打开。
- preview 编译失败或二维码未生成时，脚本退出非 0；Telegram 展示 `状态：⚠️ 失败`、退出码、stdout/stderr 摘要与详情按钮。

## 7. 风险与回滚

- 风险：`wx-auto-preview` 会多执行一次内部 `preview`，耗时会增加，但这是避免误报成功的必要校验。
- 风险：微信开发者工具 preview 可能产生一次临时预览记录；脚本不回传临时二维码。
- 回滚：移除 `scripts/gen_preview.sh` auto-preview 前置 preview 校验段，回退新增测试与 AGENTS/本文档。
