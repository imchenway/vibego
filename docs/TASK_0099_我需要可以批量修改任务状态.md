# /TASK_0099 我需要可以批量修改任务状态

## 1. 背景

用户需求：

1. 参考 `/TASK_0095` 的**批量推送任务**选择方式
2. 在任务列表中支持**批量修改任务状态**
3. 分页数量需要可以调整

本次按 YOLO 直接执行，仍严格遵循 `vibe -> design -> develop` 与 TDD 门禁。

  - `bot.py`（锚点：`_schedule_parallel_cleanup_for_done`）

## 3. vibe 结论（用户视角）

### 3.1 现状

1. 用户可从任务列表进入**批量推送任务**
2. 用户可从任务详情对单个任务切换状态
3. 任务列表虽然支持 `limit` 参数，但 Telegram 列表/批量选择界面里**没有直接调整每页数量的按钮**

### 3.2 本次新增

1. 在任务列表新增**批量修改状态**入口
2. 复用批量推送的多选体验：
   - 逐条勾选
   - 全选当前页
   - 反选当前页
   - 跨页保留已选结果
3. 统一选择一个目标状态后，按勾选顺序逐个执行
4. 普通任务列表与批量状态视图都支持切换每页数量：`10 / 20 / 50`

### 3.3 验收口径

1. 任务列表出现 `🔁 批量修改状态`
2. 进入后可多选、跨页保留、支持页级快捷选择
3. 可直接切换每页数量
4. 统一选择目标状态后：
   - 成功项更新状态
   - 已是目标状态的任务记为跳过
   - 失败项不影响其它任务继续执行
5. 若目标状态为 `done`，仍触发并行运行态异步清理
6. 执行后恢复任务列表，并输出结果摘要

## 7. 最小实现

### 7.1 任务列表

- `bot._build_task_list_view`
  - 顶部新增每页数量切换：
    - `10条/页`
    - `20条/页`
    - `50条/页`
  - 底部新增 `🔁 批量修改状态`

### 7.2 批量状态修改视图

- 复用批量推送的勾选口径：
  - 单条 toggle
  - 翻页
  - `✅ 全选当前页`
  - `🔁 反选当前页`
  - 跨页保留已选结果
- 新增批量状态视图内的每页数量切换

### 7.3 统一状态选择

- 新增 `TaskBatchStatusStates.waiting_status`
- 统一选择目标状态后逐条执行：
  - `success`：状态成功写库
  - `skipped`：已是目标状态
  - `failed`：任务不存在或 `update_task(...)` 抛错

### 7.4 done 状态兼容

- 若批量目标状态是 `done`，对成功项调用：

```python
_schedule_parallel_cleanup_for_done(updated.id)
```

保持与 `task:status:*` 单任务链路一致。

## 10. 风险与边界

1. 本次仅覆盖**任务列表入口**
   - 未扩展到搜索结果页直接进入批量状态修改
2. 本次分页数量切换仅提供固定选项：
   - `10 / 20 / 50`
3. 批量执行采用“逐条继续”策略：
   - 单条失败不会中断整个批次
