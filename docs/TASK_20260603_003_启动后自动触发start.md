# TASK_20260603_003 启动后自动触发 /start（DEVELOP）

## 1. 需求与结论

用户要求：每次启动项目时，现在会自动推送任务列表；需要改成自动触发一次 `/start`。

结论：Worker 启动后仍会对已知目标 chat 主动发一条消息，但内容从“任务列表 InlineKeyboard”改为“`/start` 等价欢迎语 + Worker 主键盘”。这样启动后只刷新主菜单与底部键盘，不再刷出任务列表。

## 2. 现状证据

| 事实 | 证据 |
| --- | --- |
| Worker 启动入口在 `main()` 中初始化服务、同步命令菜单后调用 `_broadcast_worker_keyboard(_bot)` | `bot.py`（锚点：`main`、`_broadcast_worker_keyboard(_bot)`） |
| 旧 `_broadcast_worker_keyboard` 会调用 `_build_task_list_view(...)` 并发送任务列表 | `bot.py`（锚点：`_broadcast_worker_keyboard`、`_build_task_list_view`） |
| 手动 `/start` 入口由 `on_start` 处理，回复欢迎语和 `_build_worker_main_keyboard()` | `bot.py`（锚点：`on_start`、`_build_worker_main_keyboard`） |
| 自动广播目标来自 state/projects/env 聚合 | `bot.py`（锚点：`_resolve_worker_target_chat_ids`）；`tests/test_chat_menu_buttons.py`（锚点：`test_worker_resolve_targets_reads_state_and_config`） |

## 3. 方案对比

### 方案 A：启动广播发送 `/start` 等价欢迎语（本次采用）

- 做法：抽出 `_build_worker_start_message_text(...)`，手动 `/start` 与启动广播共用文案；启动广播不再调用 `_build_task_list_view(...)`。
- 优点：最小改动；符合“自动触发一次 /start”的用户意图；不会启动时刷任务列表。
- 缺点：不是在 aiogram 内部构造伪 Message 调 handler，而是复用同一文案与键盘 helper。

### 方案 B：启动时构造伪 Message 并直接调用 `on_start`

- 优点：形式上最接近“触发 handler”。
- 缺点：aiogram `Message` 对象字段复杂，伪造成本高，容易引入测试脆弱性。

### 方案 C：只推送主键盘不发欢迎语

- 优点：更少消息。
- 缺点：不符合“触发一次 /start”，用户看不到 `/start` 的说明内容。

| `tests/test_chat_menu_buttons.py` | 是 | 将启动广播测试从“推送任务列表”改为“触发 `/start` 等价欢迎消息”。 |

| Master 生命周期 | 否 | 不改变 Master 启停状态机，只改变 worker 启动后的 Telegram 通知内容。 |
| 任务列表功能 | 否 | 手动 `/task_list`、任务列表按钮、状态流转后的自动任务列表不受影响。 |

## 5. 契约变更

1. Worker 启动后自动广播不再调用 `_build_task_list_view(...)`。
2. Worker 启动后对每个目标 chat 发送 `/start` 等价欢迎消息。
3. 自动欢迎消息携带 `_build_worker_main_keyboard()`，因此底部键盘仍会同步。
4. 手动 `/start` 与启动自动广播复用 `_build_worker_start_message_text(...)`，避免文案漂移。
5. 若存在 `ENV_ISSUES`，自动广播在欢迎消息后追加环境问题提示，与手动 `/start` 保持一致。

## 8. 风险与回滚

| 风险 | 影响 | 缓解 |
| --- | --- | --- |
| 启动后不再自动展示任务列表 | 用户需要手动打开任务列表 | 用户明确要求改为 `/start`；任务列表仍通过命令菜单/任务入口可用。 |
| 自动广播目标 chat 缺失 | 启动后无人收到 `/start` | 目标解析逻辑不改，仍复用 `_resolve_worker_target_chat_ids`。 |
| ENV_ISSUES 提示丢失 | 启动后环境问题不可见 | 自动广播在欢迎消息后追加 `_format_env_issue_message()`。 |

回滚方式：还原 `_broadcast_worker_keyboard` 为调用 `_build_task_list_view(...)` 并发送任务列表；删除新增测试断言与文档/AGENTS 约束；无数据库回滚。
