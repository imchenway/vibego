# TASK_20260708_001 AGENTS 新模板触发边界与提问门禁修正

## 背景

用户复核 `<feat> 调整提示词` 后指出三类问题：

1. `AGENTS.md` 事实表仍停留在旧模板口径，需要按当前新模板更新。
2. `vibe-diagram / HTML` 触发边界需要判断是否只是提示词精简，还是实质收窄。
3. Readiness Gate 之前的交互不稳定：仍可能一次问 2-3 个独立问题，且没有稳定优先通过提问确认用户意图、真实需求再进入设计。

## 取证结论

- 对比 `1adb35b8..72f97762`，`vibe-diagram / HTML` 不是单纯精简，而是删除了多个旧触发场景：具体对象/代码/文件更新/diff
  解释、行为/故障为什么、系统/业务/流程/时序/需求决策视觉沟通、完整逻辑/数据口径/根因链路、以及“生成或更新项目内单文件
  HTML / HTML 是主交互界面”。
- 当前模板虽然有 `Readiness Gate`，但主要写成“进入 develop 前”的清单；缺少“未达到 95%
  前唯一允许动作”的阻断式交互循环，因此模型容易先输出设计/计划，或一次性打包多个澄清问题。

## 设计决策

- 不恢复旧长提示词，采用“短模板 + 紧凑触发矩阵”。
- 新增 `Interaction Gate`，把 Readiness 未达标状态显式标记为 `READINESS: BLOCKED`。
- Readiness 未达标前，只允许只读取证、列已确认事实、列阻塞未知，并一次只问一个最高影响问题；不得输出完整设计、实现计划、任务拆解或最终方案。
- Product Design 恢复条件路由和安全边界：环境提供才按需使用、不会自动安装、缺失回退、必须先确认 brief、无视觉目标先 3
  个方向等待选择。

## 变更点

- `AGENTS-template.md`
    - `Readiness Gate` 前移到进入设计、计划或 develop 前。
    - 新增 `Interaction Gate`。
    - 恢复紧凑版 `vibe-diagram / HTML` 触发矩阵。
    - 恢复 Product Design 条件路由、brief gate 和前端实现边界。
- `AGENTS.md`
    - 更新 Facts Table 中 AGENTS 模板协作内核、具体对象解释/行为故障为什么、HTML 图形沟通、Product Design、HTML 交付后文本压缩等事实。
- `tests/test_agents_template_migration.py`
    - 新增事实表同步断言。
    - 将旧“移除 brainstorming 阶段门禁”测试改为 `Interaction Gate` 行为断言。
    - 更新可选评论断言为中文口径。
