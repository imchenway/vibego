系统架构：vibego 全貌是 Telegram 控制面 + tmux 模型工作面 + SQLite/JSONL 证据面

一张图看清入口、控制面、worker、运行中间件、状态与证据；所有节点均附源码或文档锚点。

vibego 的核心不是一个 Web 服务，而是 Telegram Bot 编排本机 tmux 中的 AI CLI；SQLite/JSONL/日志共同形成可追溯证据面。 无独立 MQ：队列语义主要由 Telegram 事件处理、tmux queued 输入、session JSONL watcher 与 SQLite 状态共同承担。

入口与依赖：README.md（功能介绍、环境依赖、目录结构、模型切换）；pyproject.toml（dependencies、project.scripts）。
控制面：master.py（MasterManager、run_worker、stop_worker、StateStore）；project_repository.py（ProjectRepository、projects 表、slug_migrations）。
Worker 工作面：bot.py（_handle_prompt_dispatch、_dispatch_prompt_to_model、tmux_send_line、tmux_queue_line、_deliver_pending_messages_locked）。
领域存储：tasks/service.py（tasks、task_notes、task_attachments、task_history、task_sequences）；command_center/service.py（commands、command_aliases、command_history）。
运行中间件：scripts/run_bot.sh、scripts/start_tmux_codex.sh、scripts/session_binder.py、scripts/models/common.sh。
状态与证据：vibego_cli/config.py（CONFIG_ROOT、PROJECTS_JSON、MASTER_DB、MASTER_STATE、LOG_FILE）；logging_setup.py（LOG_FORMAT）。
