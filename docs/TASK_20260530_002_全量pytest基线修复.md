# TASK_20260530_002 全量 pytest 基线修复

## 1. 背景与用户决策

- 用户需求：`待决策项全部按模型推荐`。
- 执行含义：沿用上一轮模型推荐的高置信度方案，先修复当前全量 `pytest` 基线红项，再谈后续发布或功能变更。
- 流程说明：当前环境存在 `comet-hotfix` skill，但本仓库未初始化 OpenSpec 变更目录（`openspec list --json` 返回 `No OpenSpec changes directory found. Run 'openspec init' first.`），因此本次按 Comet hotfix 等效路径执行：复现 -> 根因 -> TDD 修复 -> 双轮验证 -> 文档沉淀。

## 2. 现象 -> 影响 -> 根因 -> 修法 -> 验证

### 2.1 现象

全量测试存在基线红项，主要集中在四类：

1. `tests/test_agents_template_migration.py::test_enforced_notice_points_to_agents_md`：`bot.ENFORCED_AGENTS_NOTICE` 退化为仅 `以下是用户需求描述：`。
2. `tests/test_parallel_flow.py` 多个用例：`ParallelLaunchSession.__init__()` 或 `_begin_parallel_launch()` 缺少必填 `send_mode`。
3. `tests/test_plan_progress.py::test_session_ack_message_silent`：测试只关心 ack 消息静默，但当前投递确认逻辑要求 session JSONL 中出现用户输入确认，旧 fixture 未模拟模型消费。
4. `tests/test_tmux_send_line.py` 两个用例：预期提示词仍硬编码短前缀，未跟随完整规约前缀恢复。

### 2.2 影响

- 普通 Telegram 直聊进入 tmux 前的强制规约前缀可能缺失，模型不再被明确要求读取 AGENTS 与执行 TDD 门禁。
- 并行分支选择流程的旧入口/测试构造被 `send_mode` 必填破坏，形成非必要兼容性回归。
- ack 静默测试未跟随“投递确认”新契约更新，导致测试验证到了错误分支。
- 全量测试不绿，阻断后续发布与功能交付判断。

### 2.3 根因

| 根因 | 证据锚点 | 判断 |
|---|---|---|
| 规约提示词常量被压缩为短文案 | `bot.py`（锚点：`ENFORCED_AGENTS_NOTICE`）；`tests/test_agents_template_migration.py`（锚点：`test_enforced_notice_points_to_agents_md`） | 行为漂移，应恢复完整契约 |
| `send_mode` 是可选行为参数，但被放进必填构造链 | `bot.py`（锚点：`_begin_parallel_launch`、`ParallelLaunchSession`）；`tests/test_parallel_flow.py` | 兼容性回归，应默认 `None` |
| ack 静默用例未模拟 Codex 已消费用户输入 | `bot.py`（锚点：`_confirm_or_retry_prompt_delivery`）；`tests/test_plan_progress.py`（锚点：`test_session_ack_message_silent`） | 测试 fixture 过期，应补真实确认条件 |
| tmux 投递确认测试硬编码旧短前缀 | `tests/test_tmux_send_line.py`（锚点：`test_dispatch_prompt_retries_with_queue_when_user_prompt_not_confirmed`、`test_dispatch_prompt_reports_unconfirmed_after_retry`） | 测试断言应引用源常量，避免再次漂移 |

### 2.4 修法

1. 恢复 `bot.ENFORCED_AGENTS_NOTICE` 完整中文规约前缀，包含：读取全局/根目录/受影响子项目 AGENTS、继续 `vibe -> design -> develop`、PLAN/YOLO 均执行 TDD、未指定默认 PLAN、以 `以下是用户需求描述：` 分隔用户原文。
2. 将 `_begin_parallel_launch(..., send_mode=None)` 与 `ParallelLaunchSession.send_mode: Optional[str] = None` 调整为兼容旧调用的可选参数。
3. 在 `test_session_ack_message_silent` 中补充 session pointer 与 JSONL 用户输入事件，模拟 tmux 发送后模型已消费本次输入，使测试继续验证“会话确认消息应静默发送”。
4. 将 `tests/test_tmux_send_line.py` 的预期提示词改为基于 `bot.ENFORCED_AGENTS_NOTICE` 拼接，避免硬编码重复漂移。
5. 在 `AGENTS.md` Facts Table 增补“Telegram→tmux 提示词规约前缀约束”，沉淀长期证据。

## 4. 契约变更

### 4.1 对外行为契约

- 普通非 slash 用户消息投递到 tmux 前，必须自动追加完整 `ENFORCED_AGENTS_NOTICE`。
- slash 命令、已内置规约的 Plan 实现提示，不重复追加规约前缀。
- 并行启动链路中 `send_mode` 继续作为可选参数；未传入时默认为 `None`，保持历史入口兼容。

### 4.2 测试契约

- 需要验证完整规约前缀时，测试优先引用 `bot.ENFORCED_AGENTS_NOTICE` 源常量，而不是复制长字符串。
- 任何依赖普通直聊投递成功的测试，都需要满足 session JSONL 用户消息确认契约，或显式关闭确认逻辑。
