# TASK_20260310_003 并行 CLI 分支选择页显示当前分支

## 1. 背景

- 用户诉求：在“新建分支 + 新 CLI 并行处理”的分支选择页里：
  1. 显示当前分支
  2. 高亮当前分支
  3. 把当前分支排到第一位

## 2. 现状结论

- `bot.py` 旧标题仅显示：
  - 并行任务
  - 当前仓库
  - 请选择基线分支
- `bot.py` 旧按钮仅显示本地/远端分支名，没有“当前”标记
- `parallel_runtime.py` 旧 `BranchRef` 没有当前分支状态，也没有当前 branch 读取 helper

## 3. Class Impact Plan

   - `_format_parallel_branch_button_label`（新增）

### 3.3 直连依赖测试纳入依据

- `tests/test_parallel_flow.py` 是现有并行交互护栏

### 3.4 测试范围升级判断

  - 修改了共享运行时 util（`parallel_runtime.py`）

## 6. 最小实现

### 6.1 runtime 层

- `BranchRef` 新增字段：
  - `is_current: bool = False`
- 新增 `get_current_branch_state(repo_path)`
  - 正常分支：返回分支名
  - Detached HEAD：返回 `Detached HEAD`
  - 读取失败：返回 `读取失败`
- `list_branch_refs(...)`
  - 支持传入 `current_local_branch`
  - 当前本地分支排到第一位
  - 对应分支 `is_current=True`

### 6.2 UI 层

- `ParallelLaunchSession` 新增：
  - `current_branch_labels`
- `_begin_parallel_launch(...)`
  - 为每个仓库读取当前分支展示文案
- `_build_parallel_branch_title(...)`
  - 新增一行：`当前分支：xxx`
- `_format_parallel_branch_button_label(...)`
  - 新增按钮文案格式化
  - 保证 `（当前）` 尽量不被截断
- `_build_parallel_branch_keyboard(...)`
  - 当前分支按钮显示 `（当前）`

## 8. 口径与边界

1. 仅当前 **checkout 的本地分支** 标记为 `（当前）`
2. 远端同名分支不标记为当前
3. Detached HEAD 仅在标题显示，不高亮任何按钮
4. 当前分支按钮排到第一位，属于用户已确认的排序变更
