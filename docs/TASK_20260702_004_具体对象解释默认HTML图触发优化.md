# TASK_20260702_004 具体对象解释默认 HTML 图触发优化

## 背景

用户指出：“都让你解释某个事情了，不是用图解释更好吗？” 当前模板虽然已经覆盖行为/故障为什么、完整逻辑、调用链、状态流转、数据口径、前后差异和证据链，但“解释某个具体对象”的短问法仍可能被归入非视觉化轻量解释，导致“逐个解释文件更新作用”返回纯文本表格。

## 目标

- 将“解释具体对象”提升为默认图解触发边界。
- 覆盖：代码、文件更新、diff、模块、页面、接口、配置、数据、功能入口、运行结果。

- 不回到“默认所有实质沟通都 HTML”的过宽规则。
