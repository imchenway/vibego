# TASK_20260311_007 Codex 并行目录自动授权与配置回收

## 1. 背景

### 1.1 用户目标

1. 项目启动前自动检查 Codex `config.toml`，缺失 trusted 路径时自动补齐。
2. 并行 CLI 启动前自动检查 task workspace 的 trusted 路径，避免手工授权。
3. 任务完成自动清理并行目录时，同步回收对应的 trusted 配置，防止 `~/.codex/config.toml` 持续膨胀。

### 1.2 关键证据

- 并行目录当前按 task 编码派生：`bot.py`（锚点：`_parallel_runtime_root`, `_parallel_workspace_root`）
- 任务完成后已有并行目录清理链路：`bot.py`（锚点：`_delete_parallel_session_workspace`, `_cleanup_parallel_session_workspace_safely`）
- 官方文档仅公开 `projects.<path>.trust_level` 精确路径配置，未见通配符 trusted 语法：
  - <https://developers.openai.com/codex/config-reference/#configtoml>
- 本机 `~/.codex/config.toml` 已出现 task 级 workspace trusted 条目，证明路径会膨胀。

## 5. 最小实现

### 5.1 持久化层

- `parallel_runtime.py`
  - 新增 `CodexTrustedPathRecord`
  - `ParallelSessionStore.initialize()` 新增 `codex_trusted_paths` 表
  - 新增：
    - `upsert_trusted_path(...)`
    - `get_trusted_path(...)`
    - `list_trusted_paths(...)`
    - `delete_trusted_path(...)`

### 5.2 Codex config.toml 管理

- `bot.py`
  - 新增 `CODEX_CONFIG_PATH`
  - 新增 trusted scope 常量
  - 新增 `CODEX_CONFIG_LOCK`
  - 新增 config 文本 helper：
    - `_codex_project_table_header(...)`
    - `_find_codex_project_table_bounds(...)`
    - `_read_codex_project_trust_level(...)`
    - `_upsert_codex_project_trust_level_text(...)`
    - `_remove_codex_project_table_text(...)`
    - `_write_codex_config_text(...)`

### 5.3 运行时自动授权与回收

- `bot.py`
  - 新增：
    - `_ensure_codex_trusted_project_path(...)`
    - `_cleanup_codex_trusted_project_path(...)`
    - `_reconcile_codex_trusted_paths()`
    - `_ensure_primary_workdir_codex_trust()`
  - worker 启动时：
    - 先做 stale trusted 对账
    - 再确保主项目目录 trusted
  - 并行 CLI 启动前：
    - 先确保当前 task workspace trusted
  - 并行目录删除时：
    - 同步回收该 workspace 的 trusted 配置

## 7. 用户可见结果

1. worker 启动时会自动检查主项目目录的 Codex trusted 配置。
2. 并行 CLI 启动前会自动检查当前 task workspace 的 trusted 配置，不需要手工补条目。
3. 任务完成自动清理并行目录时，会同步回收对应 workspace 的 trusted 配置。
4. worker 重启时会自动清理目录已不存在的 stale trusted 条目，避免 `~/.codex/config.toml` 持续膨胀。
