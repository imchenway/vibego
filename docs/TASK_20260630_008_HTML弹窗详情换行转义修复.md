# TASK_20260630_008 HTML弹窗详情换行转义修复

## 现象

- 用户在 `docs/TASK_20260630_007_vibe-diagram系统架构图宏观拓扑示例.html` 的节点详情弹窗中看到字面量 `\n`。
- 影响：弹窗详情像代码转义字符串，不像正式 HTML 交付物，破坏阅读体验。

## 根因

- 样例 HTML 的 `data-detail` 属性把换行写成了双字符 `\n`。
- 弹窗脚本直接执行 `detail.textContent = button.dataset.detail`，浏览器不会把双字符 `\n` 自动渲染成换行。

## 修法

1. 把宏观拓扑样例的 `data-detail` 换行改为 HTML 属性内安全换行实体 `&#10;`。
2. 在弹窗渲染前增加 `normalizeDetailText()`，兜底把未来误写入的 `\\n` / `\\t` 归一化为真实换行 / 制表。
3. 更新 `vibe-diagram` skill：点击详情的换行必须使用真实换行、`&#10;` 或渲染前归一化，禁止向用户显示字面量 `\n` / `\t`
   等转义痕迹。
4. 增加回归测试，锁住宏观拓扑样例与 skill 规则。
