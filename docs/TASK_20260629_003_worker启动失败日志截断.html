故障排查：Vibego 启动失败提示被旧日志遮蔽

结论：用户看到的“最近日志”不是本次失败片段，而是 append-only 日志被旧会话和头部截断共同污染；修复后按本次 boot marker 取日志，并在超长时保留头尾。

现象启动失败提示从 2026-06-27 旧日志开始，当前 2026-06-29 根因被藏到尾部。
根因_log_tail() 无本次启动边界，截断函数只保留开头。
修法以 [run-bot] boot_id=... 为边界读取，并保留诊断尾部。
验证25 个聚焦测试通过；当前 worker 运行中，pid 63490。

影响与回滚

影响点

只改变 Master 启动失败诊断文案，不改变 worker 启动、Telegram 投递、tmux 会话或模型会话绑定逻辑。

回滚方式
