## 3. 图形设计说明

采用横向泳道图，按职责分为六段：

1. **Telegram 入口**：文本、附件/相册、任务/按钮、命令中心。
2. **入站规整**：文本聚合、附件下载/失败兜底、任务/request_input 上下文构造。
3. **统一派发**：`_handle_prompt_dispatch` 与 `_dispatch_prompt_to_model`。
4. **tmux / session**：`/plan` 预命令、queued/immediate、tmux 错误、session 绑定。
5. **事件监听**：JSONL 消费确认、快速轮询、长轮询、事件类型识别。
6. **Telegram 回传**：环境/tmux 错误、计划进度、请求用户决策、模型回复、命令结果。

判断节点均使用菱形；“是/否”标签已手工放在对应分支起点附近，避免自动布局漂移。
