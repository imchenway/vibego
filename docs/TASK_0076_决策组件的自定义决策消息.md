# /TASK_0076 决策组件的自定义决策消息

## 背景

- 用户反馈：在 `request_user_input` 的 `D. 输入自定义决策` 路径里，如果发送的是图文混合消息，当前只会把文字推给模型，附件会丢失。
- 补充反馈：Telegram 回显给用户的“决策摘要”会把自定义决策文本截断，用户要求**不能截断**。

## 关联证据

- `bot.py:11376-11445`
  - 旧逻辑仅读取 `message.text or message.caption`，未把附件并入自定义答案。
- `bot.py:8569-8591`
  - request_input 输出结构只会提交 `custom_answers` 中的字符串。
- `bot.py:11065-11091`
  - 旧摘要对自定义决策使用 `_trim_request_input_answer_preview(...)`，存在 60 字符裁切。
- `bot.py:15334-15349`
  - 媒体消息同样会进入 `_handle_request_input_custom_text_message(...)`，因此附件遗漏会真实影响图文混合输入。

## 本次实现

### 1. 自定义决策支持图文混合 / 纯附件

- 文件：`bot.py`
- 改动：
  - 自定义输入态下，若消息携带媒体附件，复用现有附件收集能力；
  - 单条媒体消息：走 `_collect_saved_attachments(...)`
  - 媒体组：走 `_collect_generic_media_group(...)`，并通过 `processed_media_groups` 避免重复消费；
  - 将 `caption/text + 附件列表` 统一拼成完整自定义答案，再写入 `session.custom_answers[...]`。

### 2. 决策摘要不再业务层截断

- 文件：`bot.py`
- 改动：
  - `_build_request_input_submission_summary(...)` 对自定义决策不再调用 `_trim_request_input_answer_preview(...)`
  - 摘要保留完整文本/附件提示

### 3. 超长摘要改走现有长文本附件兜底

- 文件：`bot.py`
- 新增：
  - `_send_request_input_submission_summary_message(...)`
- 行为：
  - 先正常发送完整决策摘要
  - 若 Telegram 返回 `message is too long`，自动改用 `reply_large_text(...)`
  - 避免“为了发得出去而先裁切业务内容”

## 结果口径

- `D. 输入自定义决策` 现在支持：
  - 纯文本
  - 单张图文
  - 纯附件
  - 媒体组图文
- 推给模型的答案仍保持原有 `answers -> [string]` 结构，不改协议，只扩充字符串内容。
- Telegram 给用户看的“决策摘要”不再业务层截断；若过长，会自动走现有附件兜底。
