三种调整方案

方案 A：只改 skill 文案

把“默认关闭候选”“证据不要抢主图”“必须有图标”继续写进 reference。成本最低，但仍依赖模型自觉，容易复发。

🌟 推荐

方案 B：规格 + 模板 + lint

先生成 DiagramSpec，再按系统架构模板渲染，最后静态检查 HTML 是否像架构图。成本中等，能把偏好变成门禁。

方案 C：完整 DSL/Renderer

把所有图型做成模板渲染器，模型只填数据。最稳，但开发成本高；建议作为 B 稳定后的第二阶段。

具体要改的 5 条规则

1

默认交付模式

普通系统架构图只出一张定稿；候选全集改为显式“校准/对比”模式。

2

架构模板库

新增总体架构、RAG 增强、Multi-Agent、端到端数据流四类 archetype。

3

图标语义表

用户、端、网关、Agent、LLM、RAG、DB、工具、人工、运维必须有固定符号。

4

证据预算

主图不展示源码路径；E#、待确认、验证命令默认进 details 或附录。

5

产物级 lint

检查 SVG/连线层、图标覆盖、节点字数、证据密度、tab 使用、纯网格退化。
