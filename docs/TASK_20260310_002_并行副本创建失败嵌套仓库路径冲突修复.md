# TASK_20260310_002 并行副本创建失败：嵌套仓库路径冲突修复

## 1. 背景

- 用户报错：

```text
并行创建失败：fatal: destination path '/Users/david/.config/vibego/runtime/parallel/hyphamall/TASK_0093/workspace/backend-java/infra-parent' already exists and is not an empty directory.
```

- 现场目录：
  - 项目工作目录：`/Users/david/hypha/mall`
  - 仓库发现结果包含：
    - `backend-java`
    - `backend-java/infra-common`
    - `backend-java/infra-parent`

## 2. 根因结论

### 2.1 仓库发现阶段包含父子重叠仓库

- `parallel_runtime.py:130-156`
  - `discover_git_repos()` 会递归扫描 `**/.git`
  - 旧逻辑未跳过“已被父仓库覆盖的嵌套仓库”

### 2.2 并行副本创建阶段缺少路径重叠校验

- `parallel_runtime.py:221-255`
  - `prepare_parallel_workspace()` 旧逻辑直接按 `relative_path` 逐个 clone
  - 当 `backend-java` 已先 clone 到 `workspace/backend-java` 后，再 clone
    `backend-java/infra-parent` 到 `workspace/backend-java/infra-parent`，会撞上非空目录

## 6. 最小实现

### 6.1 发现阶段：父仓库优先

- 在 `discover_git_repos()` 中增加相对路径层级判断：
  - 根仓库 `.` 允许与独立子仓库共存
  - 非根仓库一旦入选，就跳过其子孙仓库

### 6.2 创建阶段：防御式路径校验

- 在 `prepare_parallel_workspace()` 执行 clone 前增加 `_validate_parallel_selection_paths()`
- 若 selections 中出现非根仓库之间的相同/父子路径，提前报错：
  - `并行仓库路径重叠：... 请仅保留父仓库或子仓库其一。`

## 8. 风险与边界

1. 当前策略是“父仓库优先，跳过非根父仓库的子孙仓库”，符合本次用户确认的修复口径。
2. 根仓库 `.` 仍允许与独立子仓库共存，以兼容类似 `hyphamall` 这类“根仓库 + 多个独立子仓库”的目录形态。
3. 若未来存在“根仓库本身已追踪某个子仓库目录”的特殊布局，仍建议后续补一条专门回归测试单独治理。
