# /TASK_0077 分支选择的优化

## 1. 背景与确认口径

### 1.1 用户问题

- 现象：没有触发“共同分支批量选择”逻辑
- 诉求：若根仓库是**没有远端分支**的本地 Git 仓库，则忽略

### 1.2 已确认业务口径

1. 根仓库若为本地 Git 且 `refs/remotes` 为空，则**不参与共同分支计算**
2. 分支匹配继续保持**必须完全同名**
3. **全部仓库都参与**共同分支范围计算（除上述被忽略的根仓库外）
4. `web-base` 的 `pim-TRADE114-自采管理-社会商品库` **不视为** `TRADE114-自采管理-社会商品库`

### 1.3 关键证据

- 当前共同分支是“所有仓库求交集”：`parallel_runtime.py`（锚点：`def collect_common_branch_refs`）
- 当前会把根仓库直接纳入发现结果：`parallel_runtime.py`（锚点：`git_dir = base / ".git"`）
- 当前 `_begin_parallel_launch` 会直接拿全部发现仓库参与共同分支计算：`bot.py`（锚点：`common_branch_options = collect_common_branch_refs(...)`）
- 真实工作区复核：
  - 共发现 13 个 Git 仓库
  - 根仓库 `pim` 当前仅本地 `main`，无任何远端分支
  - 忽略该根仓库后，剩余 12 个仓库的共同分支为：`master`、`origin/develop`、`origin/master`

## 5. 最小实现

### 5.1 runtime 层

- 新增 `filter_common_branch_repo_options(...)`
- 仅忽略“没有任何远端分支的本地根仓库”
- 其它仓库继续参与共同分支交集计算

### 5.2 bot 交互层

- `_begin_parallel_launch(...)`
  - 先对 repo options 做共同分支范围过滤
  - 再执行 `collect_common_branch_refs(...)`
- `ParallelLaunchSession`

  - 增加被忽略仓库列表字段
- `_build_parallel_common_branch_title(...)`
  - 增加“共同分支计算范围：x/y 个仓库”提示
  - 增加“已忽略仓库：.（根仓库无远端分支）”提示

## 7. 用户可见结果

1. 如果根仓库只是本地 Git 且没有任何远端分支：
   - 不再阻断共同分支批量选择
2. 共同分支页会明确提示：
   - 共发现多少仓库
   - 实际参与共同分支计算的是多少仓库
   - 哪个根仓库被忽略以及原因
3. 分支名仍按**完全同名**匹配：
   - 因此 `TRADE114-...` 不会与 `pim-TRADE114-...` 合并为同一个共同分支
