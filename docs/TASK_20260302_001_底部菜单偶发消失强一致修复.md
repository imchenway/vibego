# TASK_20260302_001 底部菜单偶发消失强一致修复（DEVELOP）

## 1. 背景
- 问题：用户反馈 Telegram 底部主菜单（ReplyKeyboard）偶发消失，截图中仅剩系统 `Menu` 按钮。
- 目标：实现“强一致恢复”——在关键链路的成功/失败/自动提交失败后，都能恢复主菜单。
- 任务来源：`/TASK_0069` 子任务。

## 2. 根因分析（代码级）
### 根因 A：`全部按推荐`链路未恢复主菜单
- 位置：`on_model_quick_reply_all`。
- 现状：成功路径调用 `_send_model_push_preview(..., reply_markup=None)`，失败路径仅 `callback.answer` 弹窗。
- 影响：当用户此前处于临时键盘/隐藏状态时，链路结束后没有显式回写主菜单。

### 根因 B：request_user_input 提交收口未统一恢复
- 位置：`on_request_user_input_callback` 的 `RETRY_SUBMIT / OPTION自动提交 / SUBMIT` 分支。
- 现状：调用 `_submit_request_input_session_with_auto_retry` 时未开启 `remove_reply_keyboard=True`。
- 影响：提交成功或失败重试提示后，主菜单恢复不稳定，依赖后续其它消息“碰巧”回写。

## 3. 本次修复

### 3.1 `bot.py`
1) 修复 `全部按推荐`链路
- `on_model_quick_reply_all` 成功路径：
  - `reply_markup` 从 `None` 改为 `_build_worker_main_keyboard()`。
- `on_model_quick_reply_all` 失败路径：
  - 新增一条失败提示消息，并显式携带 `_build_worker_main_keyboard()`。

2) 统一 request_user_input 提交收口恢复
- 在以下三处调用 `_submit_request_input_session_with_auto_retry(...)` 增加：`remove_reply_keyboard=True`
  - `REQUEST_INPUT_ACTION_RETRY_SUBMIT`
  - `REQUEST_INPUT_ACTION_OPTION`（最后一题自动提交）
  - `REQUEST_INPUT_ACTION_SUBMIT`（手动提交）
- 结果：
  - 成功：决策摘要消息携带主菜单。
  - 失败：重试提示后追加主菜单恢复消息。
