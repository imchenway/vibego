# TASK_20260702_002 行为/故障为什么默认 HTML 图触发

## 背景

用户指出“点拍照/相册/拍摄/视频没反应，为什么？”这类问题明显不是普通概念问答，而是在理解具体功能行为、运行现场或故障成因。旧模板把“普通为什么/怎么做追问默认简洁文本回答”写成硬规则，容易把故障成因链路误退回聊天长文。

## 调整目标

- 不按“为什么 / 怎么做”的表面问法判断是否生成 HTML。
- 先按意图分流：
  - 概念为什么、安装升级、轻量用法说明：默认简洁文本。
  - 行为/故障为什么：默认使用 `vibe-diagram` 生成单文件 HTML 图。
- 具体功能、按钮、接口、配置、权限、构建、任务、数据或运行现场出现“为什么没反应 / 为什么失败 / 为什么没生效 / 为什么走错 / 为什么变慢 / 为什么不一致”时，视为故障成因或逻辑链路理解。
- 用户要求理解完整逻辑、调用链、状态流转、数据口径、前后差异、根因链路、证据链时，默认使用 `vibe-diagram`。
