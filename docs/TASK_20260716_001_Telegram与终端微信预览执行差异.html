故障排查：Telegram 能预览而终端命令失败的原因

两边最终调用的是同一个 gen_preview.sh，但进入脚本前的参数不同。Telegram 先由 Python 层解析出项目目录并注入 PROJECT_PATH；终端命令只有 PROJECT_BASE，因此曾走进 macOS Bash 3.2 不兼容的自动探测分支。该兼容缺陷现已在预览与上传脚本中同步修复。

核心结论 Telegram “能用”是因为绕开了故障代码；永久修复已把自动探测去重改成 Bash 3.2 支持的普通数组精确遍历。
