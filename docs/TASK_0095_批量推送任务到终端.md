# /TASK_0095 批量推送任务到终端

## 1. 背景

用户希望在 vibego 中支持**批量推送任务**：

1. 从**任务列表**发起
2. 通过**多选**确定要推送的任务
3. 本批次的会话 / 模式参数**统一只选择一次**
4. 每个任务按其当前单任务推送口径，**以排队消息（queued/Tab）依次发送到终端**

已确认决策：

- 批量入口：**任务列表多选**
- 批量范围：**勾选任务**
- 参数选择：**统一一次**

## 2. 关键证据

- 当前只支持单任务详情页 `🚀 推送到模型`：`bot.py`（锚点：`_build_task_actions`, `on_task_push_model`）
- 现有任务列表能力：`bot.py`（锚点：`_build_task_list_view`, `on_task_list_page`, `on_task_list_search`, `TaskViewState`）
- 现有“现有 CLI 会话选择”能力：`bot.py`（锚点：`_list_project_live_sessions`, `_show_push_existing_session_view`）
- 现有排队发送能力：`bot.py`（锚点：`tmux_queue_line`, `PUSH_SEND_MODE_QUEUED`）
- 单任务推送 payload 构造：`bot.py`（锚点：`_build_model_push_payload`, `_push_task_to_model`）

## 6. 最小实现

### 6.1 任务列表入口

- 在普通任务列表页底部新增：

```text
🚀 批量推送任务
```

- 点击后将当前列表消息切换到“批量推送任务”勾选视图。

### 6.2 批量勾选视图

- 每条任务按钮切换为“选中/未选中”态
- 支持分页时保留已选任务
- 底部操作：
  - `🚀 确认批量推送（N）`
  - `❌ 取消批量推送`

### 6.3 统一参数

- 本批次统一只选一次：
  - 目标会话（复用现有会话选择口径）
  - 模式（PLAN / YOLO）
- 发送方式固定为：
  - `PUSH_SEND_MODE_QUEUED`

### 6.4 批量执行

- 复用 `_push_task_to_model(...)`
- 对勾选任务按**勾选顺序**逐个调用
- 每个任务：
  - `push_mode` 统一
  - `send_mode=PUSH_SEND_MODE_QUEUED`
  - `dispatch_context` 统一
- 完成后恢复原任务列表消息，并发送批量结果摘要。

## 10. 2026-03-14 补充迭代：多选时支持全选/反选当前页

### 10.1 新增需求

用户补充要求：

1. 点击 `🚀 批量推送任务` 进入多选后，需要支持：
   - `✅ 全选当前页`
   - `🔁 反选当前页`
2. 操作口径仅作用于**当前页可见任务**
3. 其他页已选结果必须保留
4. 按钮位置放在 `🚀 确认批量推送（N）` 的上一行

### 10.2 关键证据

- 当前批量勾选页仅支持逐条点选、翻页、确认、取消：`bot.py`（锚点：`_build_task_batch_push_view`）
- 当前勾选逻辑仅支持单条 toggle：`bot.py`（锚点：`on_task_batch_push_toggle`）
- 当前分页逻辑会保留跨页已选状态：`bot.py`（锚点：`on_task_batch_push_page`）

### 10.3 本次实现

1. 新增页级快捷回调常量：
   - `TASK_BATCH_PUSH_SELECT_PAGE_CALLBACK`
   - `TASK_BATCH_PUSH_INVERT_PAGE_CALLBACK`
2. 在批量勾选页新增按钮行：
   - `✅ 全选当前页`
   - `🔁 反选当前页`
3. 新增页级 handler：
   - `on_task_batch_push_select_page`
   - `on_task_batch_push_invert_page`
4. 新增辅助逻辑：
   - `_load_task_batch_push_page`
   - `_extract_task_batch_push_selection`
   - `_list_task_batch_push_page_ids`
   - `_apply_task_batch_push_page_shortcut`

### 10.4 行为口径

- **全选当前页**
  - 仅补齐当前页可见任务
  - 不清除其他页已选任务
  - 已选任务保持幂等，不重复追加
- **反选当前页**
  - 仅对当前页可见任务做 toggle
  - 当前页已选任务取消，未选任务加入
  - 其他页已选任务保持不变
- **确认数量**
  - 继续基于全局 `selected_task_order` 统计，跨页已选结果保留
