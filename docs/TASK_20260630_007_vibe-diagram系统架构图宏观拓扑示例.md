# TASK_20260630_007 vibe-diagram 系统架构图宏观拓扑示例

## 目标

按用户要求直接画一版优化后的系统架构图样例，用来验证“系统架构图应一眼看出全局架构设计”的方向。

## 图型

系统架构图。

## 设计口径

- 首屏主画布只保留宏观拓扑，不把 Controller / Facade / Handler / DB migration 等实现清单平铺在主图。
- 主图按北向南层级展示：外部入口 → 接入 SDK → API/Gateway → Agent 编排 → 工具与外部依赖 → 状态/数据/观测 → 人工兜底。
- 三条流向显性化：主请求流、数据/知识流、兜底/异常流。
- 节点只展示职责、协议、状态和关键约束；原始证据、文件路径、行号、长说明进入点击详情。
- 底部不放证据墙，只保留轻量规则矩阵。
