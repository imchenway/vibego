## 只读诊断

源文件：`/Users/david/cckg/tcapp/Back-End/css/docs/TASK_20260630_002_业务架构图.html`

问题：当前业务架构图更像章节化卡片报告。参与方、能力层、领域对象、规则矩阵被拆成多个区块，缺少一张首屏主领域地图，用户无法一眼看出业务结构。

## 推荐方案

采用方案 B：新增“业务架构图专用骨架门禁”。

默认骨架：

`参与方边界 → 能力分层 → 领域对象关系 → 规则约束热区`

并要求：

- 首屏必须是一张主领域地图，而不是摘要卡片或章节目录。
- 能力层必须承接业务目标，不能只是功能卡片。
- 领域对象必须用拥有、产生、约束、审批、结算、归档、接续等关系连接。
- 规则必须贴到对应对象或能力旁，形成约束热区。
- 证据和长说明进入点击详情，不能把主图拆成证据卡片报告。
