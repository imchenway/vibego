建议的提示词分层

这不是删除严格门禁，而是让模型每次只看到“当前任务一定需要的规则”。官方 GPT-5.6 指南强调：每条指令只陈述一次、只暴露相关工具、每次移除一组后重跑评测。

层 1 · 模型无关短核心（全局）

证据、fail-closed、授权边界、完成声明、只读 fast path。工程目标建议 ≤ 8KiB；不放 skill 执行细则和项目历史。

层 2 · 仓库 AGENTS

稳定 Repo Map、真实命令、测试 / DoD、特殊风险、docs 索引。建议 ≤ 12–16KiB，并为嵌套项目留预算。

层 3 · docs / skill / task memory（按需）

Facts 历史、设计细则、HTML token、完整调研和一次性验证证据。被任务命中后再加载。

授权矩阵 · 建议替代重复门禁

答复 / review / 状态：只读取证后直接交付；不触发 develop gate。

修改 / 构建 / 修复

高风险未知先一次一问；Readiness 可审计后呈现 PLAN；用户明确确认 develop 才写行为文件。

外部 / 破坏性 / 扩范围动作

即使 develop 已授权，也需单独确认；prompt gate 不能代替 sandbox、approval 和凭据最小化。
