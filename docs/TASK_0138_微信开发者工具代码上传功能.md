# /TASK_0138 微信开发者工具代码上传功能（上传到微信服务）

## 1. 目标与口径

- 新增独立命令：`wx-dev-upload`
- 功能目标：将小程序代码上传到微信服务（体验版）
- 结果口径：**仅上传，不回传二维码图片**（二维码在微信开发者管理后台查看）
- 版本号策略：默认时间戳版本，支持命令参数覆盖：`wx-dev-upload --version <版本号>`
- 失败策略：自动重试 1 次
- 端口异常（缺失/不匹配）：对齐 `wx-dev-preview` 的可恢复交互（提示 + 选择/输入端口 + 自动重试）

## 2. 主要改动

### 2.1 新增上传脚本

- 文件：`scripts/gen_upload.sh`
- 能力：
  - 复用 `gen_preview.sh` 的项目探测与端口映射解析逻辑
  - 执行微信 CLI `upload` 命令上传代码
  - 默认重试 1 次（可通过环境变量调节）
  - 兼容部分 CLI 版本参数差异：`--upload-desc` 与 `--desc`
  - 输出关键日志（项目、版本、端口、成功/失败）

### 2.2 新增默认通用命令

- 文件：`command_center/defaults.py`
- 新增：
  - `name`: `wx-dev-upload`
  - `command`: `PROJECT_BASE=... bash "$ROOT_DIR/scripts/gen_upload.sh"`
  - 描述中声明 `--version` 覆盖规则与二维码查看口径

### 2.3 Bot 侧交互增强

- 文件：`bot.py`
- 改动点：
  1. 新增上传命令常量：`WX_UPLOAD_COMMAND_NAME = "wx-dev-upload"`
  2. 微信命令目录选择流程由“仅 preview”扩展为“preview + upload”
  3. 命令触发支持上传参数解析：
     - 新增 `_extract_command_args(...)`
     - 新增 `_parse_wx_upload_args(...)`（仅支持 `--version <版本号>`）
  4. 上传参数通过环境变量覆盖注入（`VERSION=...`）并贯穿：
     - 初次执行
     - 目录选择后执行
     - 端口修复重试执行
  5. 端口异常提示从固定 `wx-dev-preview` 改为按当前命令名动态展示
  6. 从脚本输出解析项目目录时，新增上传日志匹配：`[信息] 执行上传，项目：...`
