# TASK_20260310_016 并行CLI全量复刻与批量分支选择优化

## 背景
现有新建 CLI 链路只会发现部分 Git 仓库并逐仓库 `git clone`，导致嵌套 Git 仓库内容不会随并行副本一起带出；同时分支选择仅支持逐仓库操作，缺少“所有 Git 库共同分支”的批量入口。

## 本次目标
1. 并行副本创建前先完整复刻 workdir。
2. 复制时排除默认生成物目录，并叠加全部 `.gitignore` 文件交集模式。
3. 新建 CLI 先展示“共同分支批量选择”页。
4. 底部提供“逐个选择”按钮，允许退回现有逐仓库选分支流程。
5. 确认后再复刻工作目录、切分支、创建任务分支并启动并行 CLI。

## 关键实现
- `parallel_runtime.py`
  - `discover_git_repos(..., include_nested=True)` 支持返回全部嵌套 Git 仓库。
  - `collect_common_branch_refs(...)` 计算共同分支与当前分支命中数。
  - `prepare_parallel_workspace(..., source_root=...)` 改为支持“整目录复制 + 仓库内 fetch/checkout”。
- `bot.py`
  - `ParallelLaunchSession` 增加 `base_dir/common_branch_options/selection_mode`。
  - 新增共同分支分页、共同分支选择、切换逐个选择三个回调。
  - `on_parallel_branch_confirm_callback(...)` 改为把 `source_root` 传给并行副本准备逻辑。
