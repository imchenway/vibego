# TASK_20260629_002 vibego 系统架构全貌

## 1. 任务目标

用户要求“系统架构图全貌”。本轮按 `vibe-diagram` 的系统架构图规则，生成单文件 HTML，展示 vibego 的外部入口、控制面、worker 工作面、运行与中间件、状态与证据面。

## 2. 架构结论

vibego 的核心不是 Web 服务，而是 Telegram Bot 编排本机 tmux 中的 AI CLI；SQLite、JSONL、日志与附件文件共同构成证据面。仓库未发现独立 MQ，队列语义由 Telegram 事件处理、tmux queued 输入、session JSONL watcher 与 SQLite 状态共同承担。
