# TASK_20260310_011 并行分支选择流程改单消息覆盖

## 1. 背景

用户反馈：

- 进入“新建分支 + 新 CLI 并行处理”后
- 从选择分支、翻页、切到下一个仓库，到最终展示“开始并行处理”按钮
- Telegram 会不断新增消息，导致流程消息刷屏

目标：

- 从首次分支选择开始，到点击“开始并行处理”为止
- 始终优先覆盖同一条流程消息

## 5. 最小实现

### 5.1 新增统一渲染 helper

- `_render_parallel_branch_flow_message(...)`
  - 优先 `edit_text`
  - 若编辑失败，再降级为发送新消息

### 5.2 改造分支选择回调

- `on_parallel_branch_page_callback`
  - 翻页时编辑当前消息
- `on_parallel_branch_select_callback`
  - 选中某仓库分支后，进入下一仓库时编辑当前消息
  - 全部仓库选完后，展示摘要与“开始并行处理”按钮时编辑当前消息
- `on_parallel_branch_cancel_callback`
  - 取消时优先编辑当前消息
- `on_parallel_branch_confirm_callback`
  - 点击“开始并行处理”后，先把当前消息改成：
    - `正在创建并行副本并启动并行 CLI，请稍候……`
