# TASK_20260603_007 删除测试夹具里的“测试指令”

## 1. 最新结论（2026-06-04 续写）

前一轮为了替换真实中文短语，曾在 `tests/test_plan_progress.py` 中放入固定的投递确认测试哨兵。TASK_20260604_003 已继续收口：仓库不再保留该固定哨兵文本，`test_session_ack_message_silent` 改为函数内局部随机普通文本。

同时，`bot.py` 增加发送边界保护：若整条 prompt 是 vibego 内部保留测试标记样式，则 `_dispatch_prompt_to_model` 会在触达 tmux 前 fail-closed，并返回用户可见提示。

## 2. 现状与根因

| 项目 | 说明 | 证据 |
| --- | --- | --- |
| 目标测试文件 | `tests/test_plan_progress.py` 覆盖模型回复投递、ack 静默投递、pending offset 等行为。 | `tests/test_plan_progress.py`（锚点：`test_session_ack_message_silent`、`prompt_text = f"hello from delivery confirmation test`） |
| 根因 | 固定测试哨兵虽然比真实中文短语更像夹具，但仍会形成可搜索的“保留提示词”，排障时可能被误判为产品逻辑。 | 本文历史记录；TASK_20260604_003 实施记录 |
| 生产链路边界 | Telegram 普通消息与模型投递逻辑位于 `bot.py`，本轮只新增内部测试标记阻断，不改变正常用户 prompt 的投递链路。 | `bot.py`（锚点：`_dispatch_prompt_to_model`、`_is_internal_reserved_test_prompt`） |

## 3. 需求口径

### 3.1 用户/场景

- 使用者：通过搜索测试、JSONL、会话日志排查 vibego Telegram 投递链路的开发者。
- 使用时刻：排查“是否有真实 prompt 被主动发送”时。
- 业务目标：测试夹具不暴露固定保留提示词；若未来误把内部测试标记当成整条 prompt，也不会进入 tmux。

### 3.2 核心规则

1. 真实自然语言短语不得继续作为该测试用例的假 prompt。
2. 固定投递确认哨兵不得继续保留在测试、docs、生产代码、AGENTS 证据中。
3. `DummyMessage.text` 与模拟 Codex JSONL 中的 `input_text.text` 必须由同一个函数内局部变量生成。
4. 测试语义仍是“模型已消费本次输入，因此 ack 静默投递不落入未确认失败分支”。
5. 纯内部测试标记样式的整条 prompt 需要在 `_dispatch_prompt_to_model` 入口阻断，不能进入 tmux。

### 3.3 验收标准

| AC | 验收点 | 验证方式 |
| --- | --- | --- |
| AC-01 | `tests/docs/bot.py/AGENTS.md` 不再包含旧固定投递确认哨兵。 | 按用户计划中的完整 grep 执行，无结果；本文不再沉淀旧固定值 |
| AC-02 | `DummyMessage` 与手写 JSONL 模拟文本共用函数内局部随机普通文本。 | `tests/test_plan_progress.py`（锚点：`prompt_text = f"hello from delivery confirmation test`、`json.dumps(event, ensure_ascii=False)`） |
| AC-03 | 纯内部测试标记样式的整条 prompt 不进入 tmux。 | `tests/test_task_description.py`（锚点：`test_dispatch_prompt_blocks_internal_reserved_vibego_test_marker_before_tmux`） |
| AC-04 | 投递/ack/pending 聚焦测试仍通过。 | `PYTHONPATH=. python3.11 -m pytest -q tests/test_plan_progress.py -k "ack or delivery or pending"` |
| AC-05 | 普通直聊/dispatch 聚焦测试仍通过。 | `PYTHONPATH=. python3.11 -m pytest -q tests/test_task_description.py -k "direct_prompt or dispatch_prompt"` |

生产契约新增一条防护：

```text
整条 prompt 命中 vibego 内部测试标记样式 -> 阻断并返回用户可见提示，不发送 tmux
```

测试夹具契约变更：

```text

```

## 6. 风险与回滚

| 风险 | 影响 | 缓解 |
| --- | --- | --- |
| 阻断规则误伤真实用户描述 | 用户如果整条消息就是内部测试标记样式，会被阻断 | 仅 fullmatch 整条 prompt，不拦截包含该样式的普通需求描述。 |
| JSONL 模拟文本与 `DummyMessage.text` 不一致 | 投递确认测试可能误判未确认 | 两处共用 `prompt_text` 局部变量，并由 `json.dumps` 生成 JSONL。 |
| 全量测试存在既有失败 | 本轮无法声明全量绿 | 在 TASK_20260604_003 记录 baseline 与最终全量失败均为既有 `tests/test_agents_template_migration.py` 三项。 |

回滚方式：撤销 `bot.py` 内部测试标记阻断逻辑、撤销对应测试与文档更新；无 DB/配置/运行期数据回滚。
