落地结构：每一层只承担一种职责

LAYER 01触发层

什么时候必须 HTML / 什么时候触发 vibe-diagram。

文件AGENTS-template.md

只保留 HTML-first、Skill routing、交付信封、全局工作契约。不要承载具体画图细节。

边界AGENTS 管“何时”

一旦进入制图，具体图形语法、布局、箭头、节点承载交给 vibe-diagram 内核与引用文件。

收益避免 AGENTS 与 skill 双写同一规则

后续优化某类图，不需要同时改全局工作契约。

LAYER 02薄内核

进入制图后的共同协议。

保留交付 + 路由 + 共性红线

单文件 HTML、Codex/Telegram 交付、图型选择、卡片堆门禁、节点承载、可访问性与自检。

新增图型引用索引

路由到系统架构图就读取 references/system-architecture.md；业务架构图读取对应领域地图规则。

限制内核不展开每类图全部细节

内核只写“必须读取对应 reference”，避免每次生成页面设计稿时也背着故障排查细则。

LAYER 03图型引用

每类图独立优化、独立测试。

系统架构宏观拓扑 reference

保留 007 基线：北向南分层、宏观边界、主链路流线、证据进详情，不做层内卡片报告。

业务架构领域地图 reference

参与方、能力分层、领域对象关系、规则热区；内容重心是业务对象和边界，而不是文案报告。

故障/迭代现状链路 reference

先画当前代码/功能如何运行，再画根因、差异、修法和验证闭环。

方案选择

方案 做法 收益 风险 判断
A 继续单文件，只压缩重复内容 改动小，最快止血 仍会继续膨胀；每类图规则互相干扰 短期可用
