# TASK_20260530_001 底部按钮收敛到命令菜单

## 1. 任务口径

用户需求：把底部的 `会话实况`、`PLAN MODE`、`GOAL` 三个按钮移除，放到菜单里通过命令唤起。

本轮属于已有 Telegram Worker 交互的小范围调整：不新增业务 capability，不改数据库、不改外部 API、不新增依赖；核心是降低底部键盘认知负担，并保留原有能力入口。

## 2. 规约与证据

- 已读取当前仓库 `AGENTS.md`：Strict Evidence Mode、写入范围限制、TDD 门禁、docs 沉淀要求。
- Comet：当前仓库未初始化 `openspec/changes`，无法完整创建 OpenSpec change；本轮按 Comet tweak 的轻量路径降级执行，并在本任务文档中沉淀事实、测试矩阵与验证记录。
- UI/UX：按 Impeccable distill 思路处理，目标是减少底部常驻键盘的低频入口，把低频能力放入 Telegram 命令菜单。

## 3. 当前现状

- Worker 常驻键盘由 `bot.py::_build_worker_main_keyboard` 生成，原来包含：`📋 任务列表`、`📟 命令管理`、`💻 会话实况`、`🧭 PLAN MODE: ...`、`🎯 GOAL`。
- `💻 会话实况` 对应 `bot.py::on_tmux_snapshot_button`，内部调用 `_handle_terminal_snapshot_request` 展示会话列表。
- `🧭 PLAN MODE: ...` 对应 `bot.py::on_worker_plan_mode_button`，内部调用 `_handle_worker_plan_mode_toggle_request` 切换终端模式。
- `🎯 GOAL` 对应 `bot.py::on_worker_goal_button`，内部调用 `_show_goal_panel` 展示 Codex goal 管理面板。
- Telegram Bot 命令菜单由 `bot.py::BOT_COMMANDS` 和 `_ensure_bot_commands` 同步；本轮前仅包含 `start/help/goal`。

## 7. 风险与回滚

### 风险

1. 用户原先依赖底部按钮的肌肉记忆会变化，需要通过命令菜单选择 `/session_live`、`/plan_mode`、`/goal`。
2. Telegram 客户端可能短时间保留旧键盘缓存；本轮保留旧按钮 handler，点击旧按钮仍可工作，降低迁移风险。
3. `/plan_mode` 仍是直接发送 tmux 切换键；如果 tmux 会话不存在，沿用原失败提示。

### 回滚

1. 恢复 `_build_worker_main_keyboard` 中会话实况、PLAN/MODE、GOAL 三个按钮。
2. 回滚 `tests/test_chat_menu_buttons.py` 中主键盘结构与命令入口断言。
3. README/AGENTS/docs 回退到底部按钮可见口径。

## 8. 边界

- 不删除旧按钮 handler 与常量，只从主键盘移除，避免历史 Telegram 键盘残留时点击无响应。
- 不改任务列表、命令管理、会话实况页面内部结构。
- 不改 Codex `/goal` 的 session 同源、安全校验与投递确认逻辑。
