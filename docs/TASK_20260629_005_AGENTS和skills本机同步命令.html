交付验收：AGENTS 和 skills 本机同步命令

已落地：新增 /agents_sync + 系统设置按钮 + CLI，同步到配置覆盖目录，后续启动优先使用 override。

入口
/agents_sync

Telegram 命令菜单可点击；只允许管理员执行。

系统设置按钮

新增“🧩 同步 AGENTS/Skills”，与 slash 命令走同一服务。

CLI 兜底

vibego agents-sync --source-root PATH 支持终端手动执行。

源解析
D1 指定 source-root？

优先使用 MASTER/VIBEGO_AGENTS_SOURCE_ROOT。

D2 有上次成功源？

读取 CONFIG_ROOT/agents/source_root.json。

D3 兜底安装包

使用 pipx 包内 AGENTS-template 与 data/skills。

覆盖目录
原子写入 current

先写临时目录，再替换 CONFIG_ROOT/agents/current。

复制模板

AGENTS-template.md 成为后续自动同步优先源。

复制 skills

保留 SKILL.md 与 agents/*.yaml。

同步目标
Codex

~/.codex/AGENTS.md，仅替换 vibego managed block。

Claude/Gemini

~/.claude/CLAUDE.md 与 ~/.gemini/GEMINI.md。

统一规约

CONFIG_ROOT/AGENTS.md 供 worker 注入提示引用。

启动保护
run_bot.sh

优先读取覆盖目录，防止下次启动回退到 pipx 旧模板。

start_tmux_codex.sh

恢复/新建 Codex 会话时同样优先覆盖目录。

Fail-closed

源不完整或目标写入失败时列出失败项，不伪装成功。

方案对比

推荐 A 配置覆盖目录

真正支持不发版同步最新提示词/skills；后续启动不会被旧包覆盖。

B 只改目标 AGENTS

改动小，但下一次 worker 启动可能又被 pipx 旧模板覆盖。

C 远端 raw 拉取

依赖网络/认证/私有仓库策略，不适合作为默认入口。
