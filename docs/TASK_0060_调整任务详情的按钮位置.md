# TASK_0060 调整任务详情的按钮位置

## 1. 背景（现状）

在任务详情页当前操作区中：

- “查看历史” 与 “报告缺陷” 同一行展示；
- “删除（归档）” 单独占一整行，靠近底部；

用户希望调整按钮位置：将“删除（归档）”移动到“查看历史”的位置，并移除“查看历史”按钮。

参考截图：`20260203_024119928-f6f0f1d5927a.jpg`

## 2. 目标（用户视角）

在任务详情页：

1) 不再显示“查看历史”按钮；  
2) “删除（归档）”出现在原“查看历史”的位置（与“报告缺陷”同一行）；  
3) 页面内只保留一个“删除（归档）”入口，避免重复；  
4) 删除仍保留二次确认，避免误触。  

## 3. 决策结果（用户确认）

用户选择：`1A 2A 3A`

1) 删除入口：仅保留一个（位于原“查看历史”位置）  
2) 查看历史：仅移除任务详情页入口（能力/回调不强制下线）  
3) 删除交互：保留二次确认  

## 4. 验收标准（AC）

1) 任务详情页不再出现“查看历史”按钮。  
2) “删除（归档）”按钮出现在原“查看历史”的位置。  
3) 任务详情页仅保留一个“删除（归档）”入口（不重复）。  
4) 点击“删除（归档）”后仍会进入二次确认流程，确认后才执行归档。  

## 5. design（开发设计 + 用例）

### 5.1 现状定位（技术视角）

- 任务详情渲染：`bot.py` → `_render_task_detail(...)` → `_build_task_actions(task)`  
- 任务详情按钮构造：`bot.py` → `_build_task_actions(task)`  
  - 当前“查看历史”回调：`task:history:<task_id>`  
  - 当前“删除（归档）”入口回调：`task:delete_prompt:<task_id>`（进入二次确认）  
- “查看历史”回调处理：`bot.py` → `@router.callback_query(F.data.startswith("task:history:"))`  
- “删除（归档）”回调处理：`bot.py` → `@router.callback_query(F.data.startswith("task:delete_prompt:"))`  

### 5.2 实施方案（按已选决策落地）

仅调整任务详情页按钮布局（不改历史能力与回调处理逻辑）：

1) 修改 `_build_task_actions(task)`：
   - 在“报告缺陷”所在行，将右侧按钮替换为“删除（归档）”
   - 移除“查看历史”按钮
   - 移除底部单独一行的“删除（归档）”，确保仅保留一个入口
2) 保持“删除（归档）”的回调 `task:delete_prompt:<task_id>` 不变，以复用现有二次确认与归档逻辑。

### 5.3 影响范围

- 仅影响任务详情页内联按钮布局与相关用例断言（如有）。
- 不涉及数据库/任务服务接口/历史记录存储逻辑变更。

### 5.4 用例 Checklist（像测试一样）

1) 任务详情页按钮展示
   - 打开任务详情：不出现“查看历史”
   - “删除（归档）”位于“报告缺陷”同一行右侧
   - 页面中不存在第二个“删除（归档）”
2) 删除流程
   - 点击“删除（归档）”进入二次确认
   - 点击“确认删除（可恢复）”后归档成功（或提示失败原因）
3) 回归校验
   - “推送到模型”按钮仍按状态出现/消失
   - “返回任务列表”按钮正常工作
