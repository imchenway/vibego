# TASK_20260601_003 项目最近会话上下文索引

## 1. 需求结论

用户确认采用保守方案：本地维护每个项目最近 3 个已绑定模型会话的 JSONL 路径索引；新会话启动时自动把路径索引写入 Codex 提示词。该方案不复制 JSONL 文件、不写会话摘要、不默认展开 JSONL 全文。

## 2. 当前实现现状

- `scripts/session_binder.py` 负责在 worker 启动后绑定同源模型 JSONL 会话，已按 cwd、启动时间、Codex worker marker 过滤候选会话。
- `scripts/start_tmux_codex.sh` 负责生成 `codex_model_instructions.md`，并把 worker marker 写入 Codex base instructions。
- `/docs` 仍是长期上下文主来源，JSONL 只能作为近期上下文补充。

## 4. 契约变更

### 4.1 新增运行期索引文件

默认路径：

```text
~/.config/vibego/logs/<model>/<project_slug>/recent_sessions.json
```

当前 Codex worker 示例：

```text
~/.config/vibego/logs/codex/hyphafawnstudiobot/recent_sessions.json
```

索引项字段：

```json
{
  "session_id": "rollout-xxx",
  "jsonl_path": "/Users/david/.codex/sessions/.../rollout-xxx.jsonl",
  "cwd": "/Users/david/hypha/fawnStudio",
  "project_slug": "hyphafawnstudiobot",
  "bound_at": "2026-06-01T00:00:00Z"
}
```

### 4.2 保守边界

- 不复制 `jsonl_path` 指向的原文件。
- 不读取 JSONL 正文生成 summary。
- 不把 JSONL 内容默认塞入 Codex 提示词。
- 新会话提示词只注入最多 3 条路径索引。
- 模型只有在用户明确依赖“刚才、继续、上次、没反应、你刚修的”等近期上下文时，才可按需读取对应 JSONL 的尾部或关键 turn。
- `/docs` 仍是主任务上下文来源，JSONL 不能替代 docs。

## 8. 风险与回滚

### 风险

1. `recent_sessions.json` 记录的是路径，若原 JSONL 被 Codex 清理，路径可能失效；模型按需读取时需要容忍文件不存在。
2. 提示词新增路径索引会增加少量上下文，但最多 3 条，开销很低。
3. 如果运行环境没有重启 worker，新逻辑不会生效。

### 回滚

- 删除 `scripts/session_binder.py` 中 `--recent-sessions-file/--recent-limit/--project-slug` 参数与 `_update_recent_sessions_index` 相关函数。
- 删除 `scripts/start_tmux_codex.sh` 中 `RECENT_SESSIONS_FILE`、`RECENT_SESSIONS_LIMIT`、`render_recent_sessions_context` 与 binder 参数透传。
- 删除新增测试与 AGENTS Facts 行。

## 9. 最终结论
