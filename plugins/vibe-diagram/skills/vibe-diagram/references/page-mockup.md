# Page mockup reference

Use this family for screen hierarchy, page states, responsive behavior, and the user's primary path through an interface.

## Templates

- `../assets/templates/page-mockup/artboard-filmstrip.html`: compare several coherent visual directions.
- `../assets/templates/page-mockup/artboard-wireframe.html`: show navigation, viewport hierarchy, and supporting regions.
- `../assets/templates/page-mockup/primary-path-page-flow.html`: connect screens along the primary user journey.
- `../assets/templates/page-mockup/responsive-state-board.html`: compare desktop, tablet, mobile, and UI states.

Copy the selected template and preserve the artboard hierarchy instead of converting the page into a feature-card dashboard.

## Modeling rules

- Give each screen one primary objective and one dominant action.
- Separate always-visible information from progressive detail.
- Show loading, empty, error, success, and permission states when relevant.
- Keep responsive changes intentional: reflow, prioritize, collapse, or defer; do not merely scale down.
- Preserve semantic landmarks, keyboard order, focus visibility, usable targets, and reduced motion.
- Candidate artboards must differ in hierarchy or interaction model, not only color.
